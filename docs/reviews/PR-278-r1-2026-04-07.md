## HIGH — Must fix before merge

1. **`config.R` not sourced in import mirai worker — `OA_CONTENT_DOWNLOAD_COST_USD` undefined at runtime** (`R/mod_search_notebook.R:553-562`) `[claude]`
   - The batch import ExtendedTask worker (line 553) sources `db_migrations.R`, `db.R`, `interrupt.R`, `bulk_import.R`, `_ragnar.R`, `pdf.R`, `api_openalex.R`, and `import_paper.R` — but NOT `config.R`. Since the PR replaced the hardcoded `0.01` in `import_paper.R:64` with `OA_CONTENT_DOWNLOAD_COST_USD` (defined in `config.R`), the worker will throw `Error: object 'OA_CONTENT_DOWNLOAD_COST_USD' not found` at runtime. The outer `tryCatch` silently swallows the error, so cost logging is silently lost with no UI feedback. This is a regression introduced by this PR.
   - Fix: Add `source("R/config.R")` before `source("R/import_paper.R")` in the worker's source list at line 555, matching the pattern at line 525 where the reindex worker already sources `config.R`.

## MEDIUM — Should fix

2. **`all(nchar(character(0)) == 0)` returns TRUE (vacuous truth) — zero-length content vector falls into reasoning branch** (`R/api_openrouter.R:93`) `[claude]` — If `content` is `character(0)` (possible from JSON deserializers), `all()` on an empty logical is `TRUE` in R, causing the function to attempt the `$reasoning` fallback. If reasoning is also NULL, the function returns NULL, which may cause downstream errors. Fix: add `length(content) == 0 ||` before the `all()` call.

3. **NA bar heights produce invalid CSS `"height: NApx;"` in sparkline** (`R/mod_cost_tracker.R:319`) `[claude]` — When the max_ms guard passes (some valid values exist) but individual rows have `avg_latency_ms = NA`, `bar_heights[i]` is NA. `max(as.integer(NA), 2)` returns NA (no `na.rm`), and `sprintf("height: %dpx", NA)` renders `"NApx"`. Fix: add `bar_heights[is.na(bar_heights)] <- 0` after line 319.

4. **Mixed-NA sparkline test does not assert absence of `"NApx"` in rendered output** (`tests/testthat/test-cost-tracking.R:290-299`) `[claude]` — The test confirms non-null and correct class but misses the CSS defect. Add: `expect_false(grepl("NApx", as.character(result)))`.

## LOW — Nice to have

5. **API key NA guard not applied consistently** (`R/mod_search_notebook.R:2875`) `[claude]` — The embedding guard still uses the old `nchar(provider_or$api_key) == 0` pattern without an `is.na()` guard. `nchar(NA)` returns NA, which causes a warning in `if()`. The same `is.na() || !nzchar()` fix from `mod_query_builder.R:82` should be applied here.

6. **`library(shiny)` in test file is redundant/inconsistent** (`tests/testthat/test-cost-tracking.R:2`) `[claude]` — Other test files that test Shiny tag output don't explicitly load shiny. If the test harness already loads it, remove; otherwise add it to a shared helper.

7. **No test for list-of-parts with non-text parts (e.g., image)** (`tests/testthat/test-extract-message-content.R`) `[claude]` — Parts without `$text` and that aren't bare strings silently become `""` in the concatenated output. This is likely intentional but untested.
