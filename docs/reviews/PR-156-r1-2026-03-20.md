## HIGH -- Must fix before merge

1. **Missing `session` parameter in `bslib::update_switch()` for verbose_mode** (`R/mod_settings.R:327`)
   - Every other `update_switch` call in the codebase (e.g., `mod_citation_network.R:470,1032,1035`) and every `updateNumericInput` call in the same `observe()` block (lines 320, 324, 333) passes `session` explicitly. Line 327 omits it: `bslib::update_switch("verbose_mode", value = isTRUE(verbose_mode))`. Without `session`, the switch may not update correctly when settings are loaded from the database on app startup.
   - Fix: `bslib::update_switch("verbose_mode", value = isTRUE(verbose_mode), session = session)`

## MEDIUM -- Should fix

2. **Search notebook: error toast shows under still-open modal** (`R/mod_search_notebook.R:3320-3342`) -- When `generate_overview_preset()` (or similar) errors, `show_error_toast()` fires while the synthesis modal is still open. The toast appears behind/under the modal, and the modal shows "Processing response..." until `removeModal()` runs a few lines later. User sees a brief flash of confusing state. Same pattern at lines 3374, 3426. Suggestion: call `removeModal()` before `show_error_toast()` in the error handler, or restructure to always remove modal in a `tryCatch(finally = ...)` block.

3. **Duplicate `show_synthesis_modal` / `update_synthesis_status` helpers** (`R/mod_document_notebook.R:947-967`, `R/mod_search_notebook.R:472-492`) -- Identical copy-paste of both functions across two modules. If the modal layout or JS message name changes, both must be updated in sync. Suggestion: extract to a shared utility (e.g., `R/utils_synthesis.R`) or a helper module.

4. **Inconsistent error handling between document and search notebook presets** (`R/mod_document_notebook.R:998` vs `R/mod_search_notebook.R:3320`) -- Document notebook uses simple `sprintf("Error: %s", e$message)`, while search notebook uses `classify_api_error()` + `show_error_toast()` with severity-aware messaging. The search pattern is more robust. Suggestion: apply the search notebook pattern to document notebook preset handlers for consistent UX.

## LOW -- Nice to have

5. **`custom_prompt` parameter in `generate_preset()` is unused** (`R/rag.R:163-166`) -- Added as part of the refactor to `get_preset_instruction()`, but no caller passes `custom_prompt`. Dead code that adds cognitive load.

6. **CSS `:has()` selector for wizard modal positioning** (`app.R:140`) -- `.modal:has(#wizard-modal-marker)` uses `:has()`, which requires Chrome 105+, Safari 15.4+, Firefox 121+. Older browsers won't apply `margin-top: 5vh`. Consider adding the style directly to the `modalDialog()` call or using a JS fallback.
