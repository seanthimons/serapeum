## MEDIUM — Should fix

1. **N+1 DB queries in `enrich_retrieval_results()`** (`R/_ragnar.R:1165-1200`) `[claude]` — The loop makes individual `SELECT` queries per row (one for abstracts, one for documents). For a retrieval with 20+ chunks, this fires 20+ separate queries. Batch into a single query per source type using `WHERE id IN (...)` for abstracts and `WHERE filename IN (...)` for documents, then join results back to the dataframe.

2. **Error accumulation uses fragile index assignment** (`R/research_refiner.R:30,152,159,166`) `[claude]` — `errors[length(errors) + 1] <<-` inside `tryCatch` closures works but is non-idiomatic R. The standard pattern is `errors <<- c(errors, "message")`. Index-based assignment in closures can silently drop errors if the parent scope `errors` variable gets out of sync (e.g., if the loop body is later parallelized). Replace with `c()` concatenation.

3. **JSON `validate()` guard too permissive for bare strings** (`R/db.R:611-612`) `[claude]` — `jsonlite::validate(authors)` returns TRUE for valid JSON primitives like `'"Smith"'` (a JSON string). If a single author name is somehow wrapped in quotes (e.g., from a buggy upstream), it would pass through without array encoding. Tighten to also check for array/object prefix: `grepl("^\\s*[\\[\\{]", authors) && jsonlite::validate(authors)`.

4. **Refiner API error-path tests are placeholders** (`tests/testthat/test-refiner.R:156-182`) `[claude]` — Mock functions at lines 158-166 are dead code (assigned to local variables, never wired in). Tests only validate return structure with empty seeds, not actual error accumulation on API failure. Either properly mock with `local_mocked_bindings()` or rename tests to "return structure with empty seeds" to avoid false confidence.

5. **Malformed JSON test assertion too permissive** (`tests/testthat/test-rag-citations.R:71-76`) `[claude]` — Test expects `result %in% c("Unknown (2023)", "Fallback")` for malformed JSON input. This accepts two completely different behaviors without distinguishing which is correct. Pin to the actual expected output.

6. **No test for migration 019** (`migrations/019_retroactive_prompt_versions_index.sql`) `[claude]` — Migration 019 (retroactive index creation) has no dedicated test verifying it executes successfully and the index exists afterward. Test file `test-db-migrations.R` covers 018 but not 019.

## LOW — Nice to have

7. **Test comment/name mismatch** (`tests/testthat/test-db-migrations.R:205`) `[claude]` — Test is named "prompt_versions table created by migration 011" but actually tests migration 018.

8. **`setwd()` in migration test** (`tests/testthat/test-db-migrations.R:104-108`) `[claude]` — Uses `setwd(tmp_dir)` with `on.exit()` cleanup. If the test fails before `on.exit()` registers, working directory is corrupted for subsequent tests. Prefer absolute paths.
