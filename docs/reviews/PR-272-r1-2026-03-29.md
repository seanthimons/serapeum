## MEDIUM -- Should fix

1. **Unparameterized table name in test SQL helper** (`tests/testthat/test-db.R:6-10`) `[claude]`
   - `expect_columns_present()` uses `sprintf()` with `'%s'` interpolation for `table_name` in a SQL query against `information_schema.columns`. While table names are hardcoded string literals at all callsites, this is a SQL injection pattern that should not be normalized even in test code.
   - Fix: Use DBI parameterization or validate against an allowlist of known table names.

2. **Missing NULL assignment in session cleanup observer loop** (`R/mod_document_notebook.R:1882-1899`) `[claude]`
   - The `session$onSessionEnded` cleanup loop destroys observers but doesn't NULL out the references afterward. The mid-session LIFE-02 cleanup at line 939-946 correctly sets `fig_action_observers[[old_id]] <- NULL` after destroying. The session-end version omits this. In practice this is low-risk because the entire session scope is being GC'd, but it's inconsistent with the established pattern and could mask double-destroy attempts if cleanup logic is ever called outside session teardown.
   - Fix: Add `fig_action_observers[[id]] <- NULL`, `extract_observers[[id]] <- NULL`, and `delete_doc_observers[[id]] <- NULL` after each inner loop (lines 1888, 1893, 1898).

## LOW -- Nice to have

3. **Bootstrap INSERT uses literal values instead of parameterization** (`R/db_migrations.R:123-126`) `[claude]` -- `bootstrap_existing_database()` hardcodes values directly in the INSERT statement. Safe because values are literals, but inconsistent with `apply_migration()` which parameterizes both `version` and `description`.

4. **Missing edge case test for empty/NA section_filter** (`R/db.R:1112-1116`) `[claude]` -- New defensive normalization correctly filters NA/empty values from `section_filter`, but no test covers this path. A regression test would lock the behavior.
