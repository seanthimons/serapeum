CRITICAL ARCHITECTURAL NOTE:
The abstracts table does NOT store referenced_works arrays — only referenced_works_count (integer).
Backward reference analysis CANNOT use SQL unnest(). Instead, we must:
1. Get all paper_ids from the notebook's abstracts table
2. Fetch each paper's referenced_works from OpenAlex API (batch or individual)
3. Aggregate the referenced_works in R memory
This is a significant deviation from the RESEARCH.md SQL-first approach.

<task type="auto">
  <name>Task 2: Citation audit business logic + tests</name>
  <files>R/citation_audit.R, tests/testthat/test-citation-audit.R</files>
  <action>
Create R/citation_audit.R with the following functions:

1. `aggregate_backward_refs(paper_ids, email, api_key, interrupt_flag=NULL, progress_file=NULL)`:
   - Takes character vector of OpenAlex paper_ids (W-prefixed) from notebook
   - For each paper, call `get_paper(paper_id, email, api_key)` to get its referenced_works array
   - Use batch approach: process papers in chunks of 10, check interrupt between chunks
   - Collect all referenced_works URLs, strip "https://openalex.org/" prefix to get work IDs
   - Count frequency of each work_id using `table()` in R
   - Filter to only work_ids NOT in the input paper_ids set (missing papers only)
   - Write progress: "Fetching backward references: {n}/{total} papers..."
   - Return named integer vector: work_id -> count
   - Handle NULLs gracefully: papers without referenced_works contribute nothing

   NOTE: We cannot use SQL unnest() because referenced_works is NOT stored in the abstracts table.
   The get_paper() approach is needed to retrieve referenced_works from OpenAlex.
   For large notebooks (500+ papers), this will make many API calls. Use 0.1s delay between calls
   to stay within OpenAlex rate limits. Check interrupt between each chunk of 10.

   OPTIMIZATION: Use the OpenAlex batch filter approach instead of individual get_paper() calls.
   Build pipe-separated openalex_id filter: `openalex_id:W123|W456|W789` (up to 50 per request).
   Use build_openalex_request() + req_url_query(filter=..., select="id,referenced_works", per_page=200).
   This reduces API calls from N to ceil(N/50). Parse response manually (only need id + referenced_works).

2. `fetch_forward_citations(paper_ids, email, api_key, interrupt_flag=NULL, progress_file=NULL)`:
   - Takes character vector of OpenAlex paper_ids from notebook
   - Use OpenAlex filter: `cites:W123|W456|W789` (pipe-separated, max 50 per request)
   - Chunk paper_ids into batches of 50, build filter for each batch
   - For each batch, paginate through ALL results (may be thousands) using cursor pagination
   - Count how many times each citing paper appears (a paper citing 3 different notebook papers = count 3)
   - Filter to only work_ids NOT in the input paper_ids set (missing papers only)
   - Write progress: "Fetching forward citations: batch {n}/{total}..."
   - Return named integer vector: work_id -> count

   IMPORTANT: Forward citations require different approach from backward refs.
   The `cites:` filter returns papers that cite ANY of the listed works.
   We need to know WHICH notebook papers each result cites to count frequency.
   Approach: For each result, check its referenced_works array to see which notebook paper_ids it references.
   But the filter already limits to papers citing our collection, so each result cites at least one.

   Simpler approach: Just use OpenAlex `cites:` per paper (not batched by 50), then count results.
   Use get_citing_papers(paper_id, email, api_key, per_page=200) for each notebook paper.
   Collect all citing paper IDs, count frequency with table().
   Process in chunks of 10 with interrupt checks. 0.1s delay between calls.

3. `rank_missing_papers(backward_refs, forward_refs, threshold=2)`:
   - Takes two named integer vectors (work_id -> count) from steps 1 and 2
   - Merge into single data frame with columns: work_id, backward_count, forward_count, collection_frequency
   - collection_frequency = backward_count + forward_count
   - Filter to collection_frequency >= threshold
   - Sort by collection_frequency DESC
   - Return data frame

4. `fetch_missing_paper_metadata(work_ids, email, api_key, interrupt_flag=NULL, progress_file=NULL)`:
   - Takes character vector of work_ids from ranked results
   - Use batch_fetch_papers() with DOIs converted from work_ids (or use openalex_id filter directly)
   - Actually, work_ids are OpenAlex IDs (W-prefixed), not DOIs. Use build_openalex_request() with
     openalex_id filter: `openalex_id:W123|W456|...` (up to 50 per request)
   - Return list of parsed works with title, authors, year, doi, cited_by_count

5. `run_citation_audit(notebook_id, email, api_key, db_path, interrupt_flag=NULL, progress_file=NULL)`:
   - Main orchestrator function designed for mirai worker execution
   - Opens own DB connection from db_path (same pattern as run_bulk_import)
   - Steps:
     a. Get all paper_ids from abstracts table for notebook_id
     b. Create audit run via create_audit_run()
     c. Write progress: "Fetching backward references..." (step 1/3)
     d. Call aggregate_backward_refs()
     e. Check interrupt → if cancelled, save partial results and return
     f. Write progress: "Fetching forward citations..." (step 2/3)
     g. Call fetch_forward_citations()
     h. Check interrupt → if cancelled, save partial results and return
     i. Write progress: "Ranking results..." (step 3/3)
     j. Call rank_missing_papers()
     k. Fetch metadata for top 200 missing papers via fetch_missing_paper_metadata()
     l. Save results to citation_audit_results table
     m. Update audit run status to 'completed'
     n. Return summary list
   - On error: update run status to 'failed', return error info
   - On cancel: update run status to 'cancelled', save partial results

6. `import_audit_papers(work_ids, notebook_id, email, api_key, db_path, interrupt_flag=NULL, progress_file=NULL)`:
   - Takes work_ids to import from audit results
   - Fetches full paper metadata from OpenAlex using openalex_id filter
   - Stores in abstracts table using create_abstract()
   - Marks corresponding audit results as imported
   - Returns list(imported_count, failed_count)

7. `write_audit_progress(progress_file, step, total_steps, detail_message)`:
   - Simple wrapper: writes "step|total_steps|detail_message" to progress file
   - Read with `read_audit_progress(progress_file)` -> list(step, total_steps, message)
