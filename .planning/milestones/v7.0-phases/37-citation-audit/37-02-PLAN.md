Create R/mod_citation_audit.R with mod_citation_audit_ui() and mod_citation_audit_server().

**UI (mod_citation_audit_ui):**

Header section:
- Page title: "Citation Audit" with magnifying-glass-chart icon
- Brief description text: "Find frequently-cited papers missing from your collection"

Controls row:
- selectInput for notebook selection (ns("notebook_id")) — populated server-side with search notebooks only (type == "search")
- actionButton(ns("run_audit"), "Run Analysis", icon=icon("play"), class="btn-primary")
- Conditional: if cached results exist, show "Last analyzed: {date}" text and "Re-run" label on button

Results section (conditionally shown when results exist):
- Summary cards row: total papers analyzed, backward refs found, forward citations found, missing papers found
- Sortable table with columns:
  - Checkbox column for batch selection (ns("select_{work_id}"))
  - Title (truncated to ~80 chars with tooltip for full)
  - Authors (first author + et al.)
  - Year
  - Backward Count (how many collection papers cite this)
  - Forward Count (how many times this cites collection papers)
  - Collection Frequency (backward + forward, bold)
  - Global Citations (from OpenAlex cited_by_count)
  - Import button or "Imported" badge
- Sort controls: radioButtons(ns("sort_by")) with options: "Collection Frequency", "Global Citations", "Year"
  Default: Collection Frequency descending
- Batch import bar (shown when checkboxes selected):
  - "Select All" checkbox
  - "{N} selected" text
  - actionButton(ns("batch_import"), "Import Selected", class="btn-success", icon=icon("download"))

Import behavior:
- Single import button per row: actionButton with ns("import_{work_id}"), class="btn-sm btn-outline-success"
- On click: immediate action (no confirmation per user decision)
- Batch import: show confirmation dialog ("Import {N} papers?") per user decision
- After import: mark row as imported (badge), refresh results

Warning banner (shown when results are partial):
- "Results may be incomplete — analysis was cancelled or encountered errors"

**Server (mod_citation_audit_server):**

Parameters: `id, con, config_r, db_path`

Key server components:

1. Notebook list reactive:
   ```r
   notebooks <- reactive({
     notebooks <- list_notebooks(con)
     notebooks[notebooks$type == "search", ]
   })
   ```
   Update selectInput choices when notebooks change.

2. Cached results loading:
   - When notebook_id changes, check for latest audit run via get_latest_audit_run()
   - If exists, load results from get_audit_results() and call check_audit_imports() to refresh imported flags
   - Show last_analyzed timestamp

3. ExtendedTask for async audit:
   ```r
   audit_task <- ExtendedTask$new(function(notebook_id, email, api_key, interrupt_flag, progress_file, db_path, app_dir) {
     mirai::mirai({
       source(file.path(app_dir, "R", "interrupt.R"))
       source(file.path(app_dir, "R", "config.R"))
       source(file.path(app_dir, "R", "db.R"))
       source(file.path(app_dir, "R", "api_openalex.R"))
       source(file.path(app_dir, "R", "citation_audit.R"))
       run_citation_audit(notebook_id, email, api_key, db_path, interrupt_flag, progress_file)
     }, notebook_id=notebook_id, email=email, api_key=api_key,
        interrupt_flag=interrupt_flag, progress_file=progress_file,
        db_path=db_path, app_dir=app_dir)
   })
   ```

4. Run audit handler (observeEvent on run_audit):
   - Create interrupt flag and progress file
   - Show progress modal (pattern from mod_bulk_import.R):
     - Title: "Analyzing Citations" with spinner icon
     - Progress bar (striped, animated)
     - Step message text: "Initializing..."
     - Cancel button (btn-warning)
   - Invoke audit_task

5. Progress polling (observe with invalidateLater(500)):
   - Read progress file via read_audit_progress()
   - Map step numbers to progress percentages:
     Step 1/3 (backward refs): 5-33%
     Step 2/3 (forward citations): 34-66%
     Step 3/3 (ranking + metadata): 67-95%
     Complete: 100%
   - Send custom JS message "updateAuditProgress" with bar_id, msg_id, percent, message

6. Task completion handler:
   - On success: close modal, load results, show notification
   - On error: close modal, show error notification
   - Refresh audit results reactive

7. Cancel handler:
   - signal_interrupt() on cancel button
   - Results will be partial (handled by run_citation_audit returning partial)

8. Single import handler:
   - Use observeEvent with pattern matching on input IDs starting with "import_"
   - For each import click:
     a. Get work_id from input ID
     b. Call import_audit_papers(work_ids=work_id, notebook_id, email, api_key, db_path) — this is quick enough to run synchronously for single papers
     c. Mark result as imported in UI
     d. Show success notification

9. Batch import handler:
   - On batch_import click:
     a. Collect all selected work_ids from checkboxes
     b. Show confirmation: modalDialog("Import {N} papers into {notebook_name}?")
     c. On confirm: run import_audit_papers() via ExtendedTask for large batches (>10), synchronously for small
     d. Refresh results, navigate to notebook per user decision

10. Select all handler:
    - Checkbox that toggles all row checkboxes
    - Track selected count in reactive

11. Results table rendering:
    - Use renderUI to build HTML table rows (same pattern as search notebook results)
    - Each row has: checkbox, title (as link if DOI available), authors, year, backward_count, forward_count, collection_frequency (bold), cited_by_count, import button/badge
    - Sort by current sort_by selection
    - "Imported" badge (green) replaces import button for already-imported papers

Create www/js/audit-progress.js:
```javascript
Shiny.addCustomMessageHandler('updateAuditProgress', function(message) {
  var bar = document.getElementById(message.bar_id);
  var msg = document.getElementById(message.msg_id);
  if (bar) {
    bar.style.width = message.percent + '%';
    bar.textContent = message.percent + '%';
    bar.setAttribute('aria-valuenow', message.percent);
  }
  if (msg) {
    msg.textContent = message.message;
  }
});
```
