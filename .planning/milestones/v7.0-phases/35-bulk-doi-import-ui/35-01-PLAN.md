Add two new tables to `init_schema()` in R/db.R, after the existing `cost_log` table creation:

```sql
CREATE TABLE IF NOT EXISTS import_runs (
  id VARCHAR PRIMARY KEY,
  notebook_id VARCHAR NOT NULL,
  name VARCHAR NOT NULL,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  total_count INTEGER NOT NULL DEFAULT 0,
  imported_count INTEGER NOT NULL DEFAULT 0,
  failed_count INTEGER NOT NULL DEFAULT 0,
  skipped_count INTEGER NOT NULL DEFAULT 0,
  FOREIGN KEY (notebook_id) REFERENCES notebooks(id)
);

CREATE TABLE IF NOT EXISTS import_run_items (
  id VARCHAR PRIMARY KEY,
  run_id VARCHAR NOT NULL,
  doi VARCHAR NOT NULL,
  status VARCHAR NOT NULL,
  error_reason VARCHAR,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  FOREIGN KEY (run_id) REFERENCES import_runs(id)
);
```

Status values for import_run_items: 'success', 'not_found', 'api_error', 'malformed', 'duplicate'.

Add these helper functions after the existing `delete_abstract` function:

1. `create_import_run(con, notebook_id, name, total_count)` — INSERT into import_runs, return id (uuid)
2. `update_import_run_counts(con, run_id, imported_count, failed_count, skipped_count)` — UPDATE counts
3. `create_import_run_item(con, run_id, doi, status, error_reason = NULL)` — INSERT into import_run_items, return id
4. `get_import_runs(con, notebook_id)` — SELECT all runs for notebook, ORDER BY created_at DESC
5. `get_import_run_items(con, run_id)` — SELECT all items for a run
6. `delete_import_run(con, run_id)` — DELETE run and its items (items first due to FK)
7. `get_failed_import_items(con, run_id)` — SELECT items WHERE status IN ('not_found', 'api_error') for retry

Follow existing patterns: use uuid::UUIDgenerate() for IDs, DBI::dbExecute for writes, DBI::dbGetQuery for reads, parameterized queries with ? placeholders.

<task type="auto">
  <name>Task 2: Create bulk_import.R with business logic</name>
  <files>R/bulk_import.R, tests/testthat/test-bulk_import.R</files>
  <action>
Create `R/bulk_import.R` with the following functions:

1. **`extract_dois_from_bib(bib_text)`** — Extract DOIs from BibTeX content
   - Input: character vector of lines from a .bib file (from readLines)
   - Collapse to single string, use regex: `doi\s*=\s*[{"]?\s*(10\.[^},"\\s]+)` (case-insensitive, perl)
   - Extract DOI portion, clean trailing braces/quotes/commas
   - Return: list with `$dois` (character vector of extracted DOIs) and `$entries_without_doi` (integer count of @type entries that had no DOI field)
   - Count total @article/@inproceedings/@book/etc entries to determine entries_without_doi

2. **`get_notebook_dois(con, notebook_id)`** — Get existing DOIs in a notebook
   - Query: `SELECT DISTINCT doi FROM abstracts WHERE notebook_id = ? AND doi IS NOT NULL AND doi != ''`
   - Return lowercase character vector via `tolower(result$doi)`

3. **`estimate_import_time(doi_count, batch_size = 50, delay_per_batch = 0.1, api_time_per_batch = 1.5)`**
   - Calculate: n_batches * (api_time_per_batch + delay_per_batch)
   - Return formatted string: "~N seconds" if < 60s, "~N minutes" otherwise

4. **`run_bulk_import(dois, notebook_id, email, api_key, con, run_name, interrupt_flag = NULL, progress_file = NULL)`** — Main orchestration function
   - This is the function that runs INSIDE the mirai worker. It must:
     a. Open its own DB connection (mirai workers can't share connections): `con_worker <- get_db_connection()`
     b. Create import run: `create_import_run(con_worker, notebook_id, run_name, length(dois))`
     c. Call `batch_fetch_papers(dois, email, api_key, progress_callback = callback)` where callback writes progress to progress_file using a simple format: `"batch|total_batches|found|failed|message"` via writeLines
     d. Check `check_interrupt(interrupt_flag)` in the progress callback — if TRUE, stop processing
     e. For each returned paper: call `create_abstract(con_worker, ...)` to persist, then `create_import_run_item(con_worker, run_id, doi, "success")`
     f. For each error: call `create_import_run_item(con_worker, run_id, doi, error$reason, error$details)`
     g. Update import run counts: `update_import_run_counts(con_worker, run_id, imported, failed, skipped)`
     h. Close worker connection: `close_db_connection(con_worker)`
     i. Return: list(run_id, imported_count, failed_count, skipped_count, cancelled = TRUE/FALSE)
   - IMPORTANT: The `con` parameter is for the DB path — pass `attr(con, "dbname")` or the db_path from app config. The worker creates its own connection from this path.
   - Actually, simpler approach: accept `db_path` parameter instead of `con`. The caller extracts the path.

5. **`read_import_progress(progress_file)`** — Read import-specific progress
   - Parse format: "batch|total_batches|found|failed|message"
   - Return list(batch, total_batches, found, failed, message, pct) where pct = round(batch/total_batches * 100)

6. **`write_import_progress(progress_file, batch, total_batches, found, failed, message)`** — Write import progress
   - Format: paste(batch, total_batches, found, failed, message, sep = "|")
   - Use tryCatch like existing write_progress pattern
