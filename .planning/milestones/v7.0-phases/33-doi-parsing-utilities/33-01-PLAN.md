<feature>
  <name>Batch DOI Parsing with Categorized Errors</name>
  <files>R/utils_doi.R, tests/testthat/test-utils_doi.R</files>
  <behavior>
    parse_doi_list(input) accepts string or character vector, returns list with 3 components:

    Input handling:
    - Single string → split on newlines and commas ([\n,]+ regex)
    - Character vector → use as-is
    - Pre-process: trim whitespace, strip trailing periods/commas/semicolons, drop empty lines silently

    Normalization pipeline per entry:
    1. URL decode (utils::URLdecode) — handles %2F etc.
    2. Strip query params (?...) — handles browser copy-paste
    3. Existing normalize_doi_bare() — strips doi.org/dx.doi.org/doi: prefixes, lowercases

    Output:
    - $valid: character vector of unique normalized bare DOIs
    - $invalid: data.frame(original, reason) where reason is one of: missing_prefix, invalid_registrant, empty_suffix, unrecognized_format
    - $duplicates: data.frame(doi, count) for DOIs appearing 2+ times

    Test cases (input -> expected):
    - "10.1234/abc" → $valid = "10.1234/abc", $invalid empty, $duplicates empty
    - "https://doi.org/10.1234/ABC" → $valid = "10.1234/abc"
    - "10.1234/abc\n10.5678/xyz" → $valid = c("10.1234/abc", "10.5678/xyz")
    - "10.1234/abc, 10.5678/xyz" → $valid = c("10.1234/abc", "10.5678/xyz")
    - "doi: 10.1234/abc" → $valid = "10.1234/abc"
    - "invalid-string" → $invalid = data.frame(original="invalid-string", reason="missing_prefix")
    - "10.1234/abc\n10.1234/abc" → $valid = "10.1234/abc", $duplicates = data.frame(doi="10.1234/abc", count=2)
    - "10.1234/abc." → $valid = "10.1234/abc" (trailing period stripped)
    - "" or whitespace-only lines → silently ignored
    - "https://doi.org/10.1234%2Fabc" → $valid = "10.1234/abc" (URL decoded)
    - "https://doi.org/10.1234/abc?ref=pdf" → $valid = "10.1234/abc" (query params stripped)
    - c("10.1234/abc", "10.5678/xyz") → character vector input works
    - "10.1234/ABC\n10.1234/abc" → $valid = "10.1234/abc", $duplicates reports 2 (case-insensitive dedup)
  </behavior>
  <implementation>
    Enhance normalize_doi_bare() to handle URL encoding and query params (add URLdecode + query param strip before existing logic).

    Add three new functions to R/utils_doi.R:
    - split_doi_input(input): splits single string on [\n,]+ regex, trims whitespace
    - categorize_doi_error(doi_string): returns one of 4 error reason strings based on what's wrong
    - parse_doi_list(input): orchestrator — splits, pre-processes, normalizes via normalize_doi_bare(), categorizes errors, deduplicates, returns structured list

    Use base R only (strsplit, grepl, gsub, table, sapply). No new dependencies.
  </implementation>
</feature>
