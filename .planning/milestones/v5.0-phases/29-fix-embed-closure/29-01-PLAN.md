# Plan 29-01: Fix Embed Function Attachment Across All Ragnar Store Entry Points

## Goal

Fix the critical bug where ragnar's serialized embed closure loses its environment on deserialization, breaking all embedding operations. The fix: attach the embed function at runtime via the S7 `@embed` property after every store create/connect call.

## Context

**Root cause:** `ragnar_store_create()` serializes the `embed` function to disk. When `ragnar_store_connect()` deserializes it, the closure environment is completely lost — even base R functions like `runif` become unresolvable.

**Confirmed fix:** The S7 `store@embed` property is writable. We can set it after connection, bypassing the broken serialized version.

**Entry points that need fixing:**
1. `get_ragnar_store()` (line 146) — used for store creation AND connection when embedding
2. `ensure_ragnar_store()` (line 302) — directly calls `ragnar::ragnar_store_connect()` on line 308, bypassing `get_ragnar_store`
3. `rebuild_notebook_store()` (line 531) — calls `get_ragnar_store()`, so covered by fix #1

**Entry points that DON'T need fixing:**
- `connect_ragnar_store()` (line 187) — retrieval-only, read-only, no embedding needed
- `check_store_integrity()` (line 356) — test-and-disconnect, no embedding needed

## Requirements Addressed

- **EMBED-01**: Fix ragnar embed closure serialization bug
- **EMBED-02**: Verify embedding round-trip end-to-end

## Tasks

### Task 1: Create `make_embed_function()` helper

**File:** `R/_ragnar.R`
**Location:** Before `get_ragnar_store()` (around line 140)

Create a pure helper that builds the embed function without relying on closures that will be serialized:

```r
#' Build embed function for OpenRouter
#'
#' Creates a self-contained embed function that calls the OpenRouter
#' embeddings API directly via httr2. This function is set on ragnar
#' stores at runtime via the S7 @embed property, bypassing ragnar's
#' closure serialization which loses environment references.
#'
#' @param api_key OpenRouter API key
#' @param embed_model Embedding model ID
#' @return Function(texts) -> matrix of embeddings
make_embed_function <- function(api_key, embed_model) {
  force(api_key)
  force(embed_model)
  function(texts) {
    result <- get_embeddings(api_key, embed_model, texts)
    do.call(rbind, result$embeddings)
  }
}
```

Note: This function is called at runtime and its result is set on `store@embed` — it is never serialized by ragnar. The `force()` calls ensure the arguments are evaluated immediately.

### Task 2: Fix `get_ragnar_store()` — attach embed after both create and connect

**File:** `R/_ragnar.R`
**Lines:** 146-178

Replace the current implementation:

```r
get_ragnar_store <- function(path = "data/serapeum.ragnar.duckdb",
                              openrouter_api_key = NULL,
                              embed_model = "openai/text-embedding-3-small") {
  dir.create(dirname(path), showWarnings = FALSE, recursive = TRUE)

  # Require API key (needed for embed function on both create and connect)
  if (is.null(openrouter_api_key) || nchar(openrouter_api_key) == 0) {
    stop("OpenRouter API key required to create/open ragnar store for embedding")
  }

  if (file.exists(path)) {
    store <- ragnar::ragnar_store_connect(path)
  } else {
    # Create with a dummy embed (ragnar requires one at creation)
    # The real embed is attached below via @embed
    store <- ragnar::ragnar_store_create(
      path,
      embed = function(texts) stop("placeholder — should be replaced at runtime"),
      version = 1
    )
  }

  # Attach working embed function at runtime (bypasses serialization)
  store@embed <- make_embed_function(openrouter_api_key, embed_model)
  store
}
```

**Key changes:**
- API key is now required for both paths (create and connect) since we always need to attach embed
- The dummy embed passed to `ragnar_store_create()` exists only to satisfy ragnar's constructor — it's immediately overwritten
- `store@embed` is set after both paths, ensuring a working function regardless of serialization

### Task 3: Fix `ensure_ragnar_store()` — attach embed on connect path

**File:** `R/_ragnar.R`
**Lines:** 306-309

The current code at line 308 calls `ragnar::ragnar_store_connect()` directly, returning a store with a broken deserialized embed. Fix:

```r
  # If store exists, connect and attach embed function
  if (file.exists(store_path)) {
    store <- ragnar::ragnar_store_connect(store_path)
    # Attach working embed function (bypasses broken serialized closure)
    if (!is.null(api_key) && nchar(api_key) > 0) {
      store@embed <- make_embed_function(api_key, embed_model)
    }
    return(store)
  }
```

**Key change:** After connecting to existing store, attach the embed function if credentials are available. This is the path that was silently broken — the store connected fine but couldn't embed.
