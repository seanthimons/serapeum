**2. Fix MutationObserver repositioning in R/mod_citation_network.R (lines 773-836):**

Replace the `repositionTooltip` function with corrected coordinate math. The key bugs to fix:

a) **Left edge clamping is always false:** `if (cRect.left + left < cRect.left)` simplifies to `if (left < 0)` which is close but the condition as written will never trigger because `cRect.left + left` is always `>= cRect.left` when `left >= 0`. Fix: properly compute viewport position of tooltip left edge.

b) **Coordinate system:** `tooltip.style.left/top` are relative to the vis-network canvas parent, but `getBoundingClientRect()` returns viewport coordinates. The overflow math must account for the canvas position offset.

Replace the entire `repositionTooltip` function with:
```javascript
function repositionTooltip(tooltip, container) {
  requestAnimationFrame(function() {
    var tipRect = tooltip.getBoundingClientRect();
    var cRect = container.getBoundingClientRect();

    var left = parseInt(tooltip.style.left, 10) || 0;
    var top = parseInt(tooltip.style.top, 10) || 0;

    // Tooltip viewport position = canvas origin + canvas-relative offset
    var tipViewportLeft = cRect.left + left;
    var tipViewportTop = cRect.top + top;

    // Check right overflow
    var rightOverflow = (tipViewportLeft + tipRect.width) - cRect.right;
    if (rightOverflow > 0) {
      left = left - rightOverflow - 8;
    }

    // Check left overflow (recalculate after right adjustment)
    if (cRect.left + left < cRect.left) {
      left = 8;
    }

    // Check bottom overflow
    var bottomOverflow = (tipViewportTop + tipRect.height) - cRect.bottom;
    if (bottomOverflow > 0) {
      top = top - tipRect.height - 20;
    }

    // Check top overflow (after bottom flip)
    if (cRect.top + top < cRect.top) {
      top = 8;
    }

    tooltip.style.left = left + 'px';
    tooltip.style.top = top + 'px';
  });
}
```

The key insight: `cRect.left + left < cRect.left` in viewport terms checks `left < 0` in canvas-relative terms. After the right-overflow adjustment, `left` CAN become negative, so the check is valid — but the original code placed it BEFORE any adjustment could make left negative. The fix moves the check to after the right-overflow correction and uses `8` (margin) instead of `0`.
