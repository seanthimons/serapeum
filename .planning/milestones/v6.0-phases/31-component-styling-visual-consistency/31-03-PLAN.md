Add the following CSS rules inside catppuccin_dark_css(), appended before the closing of the paste0() string:

1. **Value box text override (UAT tests 10+11):** Sass compiles .bg-primary and .bg-success text color to black at build time. Dark mode updates backgrounds via CSS vars to bright Catppuccin pastels but text stays black. Per CONTEXT.md, semantic colors are used as "background fills with dark text" — but the Sass-frozen text is #000 which is too harsh. Override to use MOCHA$crust (#11111b) for softer contrast that stays readable on pastel backgrounds:

```css
[data-bs-theme="dark"] .bg-primary,
[data-bs-theme="dark"] .bg-success,
[data-bs-theme="dark"] .bg-danger,
[data-bs-theme="dark"] .bg-warning,
[data-bs-theme="dark"] .bg-info {
  color: #11111b !important;  /* Mocha Crust — dark text on bright pastel bg */
}
```

Also target value_box specifically since bslib may use different selectors:
```css
[data-bs-theme="dark"] .value-box.bg-primary .value-box-title,
[data-bs-theme="dark"] .value-box.bg-primary .value-box-value,
[data-bs-theme="dark"] .value-box.bg-success .value-box-title,
[data-bs-theme="dark"] .value-box.bg-success .value-box-value,
[data-bs-theme="dark"] .value-box.bg-warning .value-box-title,
[data-bs-theme="dark"] .value-box.bg-warning .value-box-value,
[data-bs-theme="dark"] .value-box.bg-danger .value-box-title,
[data-bs-theme="dark"] .value-box.bg-danger .value-box-value,
[data-bs-theme="dark"] .value-box.bg-info .value-box-title,
[data-bs-theme="dark"] .value-box.bg-info .value-box-value {
  color: #11111b !important;
}
```

NOTE: bslib value_box with theme="primary" generates class "bg-primary" on the container. Check the actual rendered HTML class if uncertain — but the above selectors cover both standard Bootstrap and bslib value_box patterns.

   ```

     fill = FALSE,
   ```

2. **Splash page visible card rectangle (UAT test 12):** In app.R around line 933, the welcome page card has `class = "border-0"` which removes border but NOT background. In dark mode, --bs-card-bg (#313244) contrasts against --bs-body-bg (#1e1e2e), showing a visible lighter rectangle. Change:
   ```r
   card(
     class = "border-0",
   ```
   to:
   ```r
   card(
     class = "border-0 bg-transparent",
   ```

The thematic package is already in renv.lock.
