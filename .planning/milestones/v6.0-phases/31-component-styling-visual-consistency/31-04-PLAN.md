**Gap 1 - Value box text readability:**

In `R/mod_cost_tracker.R` line 85, the subtitle p tag uses `class="small text-muted mb-0"`. The `text-muted` class renders as grey (#6c757d) which has poor contrast on the green `bg-success` background (#a6e3a1 in dark mode).

Change line 85 from:
```r
p(class = "small text-muted mb-0",
```

To:
```r
p(class = "small mb-0",
```

This removes `text-muted`, allowing the text to use the default Mocha Crust color (#11111b) set by the existing value box override in theme_catppuccin.R (added in Plan 31-03).

**Gap 4 - Disclaimer contrast:**

In `R/theme_catppuccin.R` line 186, the alert-warning dark mode background uses `rgba(249, 226, 175, 0.15)` — 15% opacity is too subtle on the Mocha Base background (#1e1e2e), appearing grey instead of yellow.

Change line 186 from:
```r
background-color: rgba(249, 226, 175, 0.15);
```

To:
```r
background-color: rgba(249, 226, 175, 0.22);
```

Also increase border opacity on line 187 from:
```r
border-color: rgba(249, 226, 175, 0.3);
```

To:
```r
border-color: rgba(249, 226, 175, 0.5);
```

This increases contrast while maintaining subtle pastel aesthetic. 22% opacity provides perceptible yellow tint without overwhelming the content.

**Gap 5 - Chat message readability:**

Document notebook assistant messages (R/mod_document_notebook.R line 706) use `class="bg-white border p-2 rounded chat-markdown"`. The existing dark mode safety net in theme_catppuccin.R covers `.bg-light` but NOT `.bg-white`.

In dark mode, `bg-white` keeps the white background but text color remains faded because there's no override forcing readable text.

Add the following CSS rule to `catppuccin_dark_css()` function in `R/theme_catppuccin.R`, immediately after the existing `.bg-light` and `.text-dark` safety net rules (around line 182):

```r
/* Override .bg-white in dark mode (document notebook chat uses this) */
[data-bs-theme='dark'] .bg-white {
  background-color: var(--bs-secondary-bg) !important;
  color: var(--bs-body-color) !important;
}
```

This treats `bg-white` the same as `bg-light` — overrides to semantic dark backgrounds with proper text contrast. Uses `!important` to override Bootstrap's compiled white background.

**Location guidance:** Insert after line 182 (the `.text-dark` rule) and before line 184 (the alert-warning comment). This keeps all safety net overrides together.
  </action>
