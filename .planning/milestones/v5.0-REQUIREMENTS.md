# v5.0 Requirements — Fix Document Embeddings

## Milestone Goal

Fix the critical bug that prevents document embeddings from working. After ragnar serializes the embed closure to disk, deserialization loses the closure environment, making `get_embeddings` (and even base R functions) unresolvable. Documents upload but never get embedded (brain icon never appears).

## Requirements

### EMBED-01: Fix ragnar embed closure serialization bug
**Priority:** Critical
**Source:** docs/plans/2026-02-22-ragnar-embedding-closure-bug.md

The embed function passed to `ragnar_store_create()` is serialized to disk. When `ragnar_store_connect()` deserializes it, the closure environment is lost — all external references fail.

**Fix approach:** Set the embed function on the S7 store object (`store@embed`) after every `ragnar_store_connect()` call, bypassing serialization entirely. The embed function must also be set after `ragnar_store_create()` returns (since the stored version will be broken on next connect).

**Acceptance criteria:**
1. PDF upload to document notebook produces embedded chunks (brain icon appears)
2. "Embed Papers" button in search notebooks successfully embeds abstracts
3. Async re-index via mirai workers successfully embeds (cross-process path)
4. Existing stores that were created before the fix can be connected and used for new embeddings
5. No regression in RAG chat (search_chunks_hybrid still returns results)

### EMBED-02: Verify embedding round-trip end-to-end
**Priority:** High

**Acceptance criteria:**
