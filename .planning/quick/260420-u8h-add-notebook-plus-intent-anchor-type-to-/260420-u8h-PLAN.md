<objective>
Add a 5th anchor type "Notebook + Intent" to the Research Refiner that combines
notebook-as-seeds (from the existing "From Notebook" type) with a user-provided
research intent (from the existing "Research Intent" / "Seeds + Intent" types).

Purpose: Users with an existing notebook of papers want to find MORE papers that
are related to that notebook AND match a specific research direction. Currently
they must choose between using a notebook as seeds OR providing intent text --
this new type lets them do both.

Output: Modified `R/mod_research_refiner.R` with the new anchor type wired end-to-end.
</objective>

In `mod_research_refiner_server` (lines 170-1006), make these changes:

1. **Validation block** (lines 398-417): Add validation for the new type. After the `if (anchor_type == "notebook_anchor")` block (line 411-417), add:
   ```r
   if (anchor_type == "notebook_intent") {
     anchor_nb_id <- input$anchor_notebook_id
     if (is.null(anchor_nb_id) || nchar(anchor_nb_id) == 0) {
       showNotification("Please select an anchor notebook.", type = "warning")
       return()
     }
     if (is.null(input$anchor_intent) || nchar(trimws(input$anchor_intent)) == 0) {
       showNotification("Please enter a research intent.", type = "warning")
       return()
     }
   }
   ```

2. **Candidate fetching block** (lines 422-472): The `notebook_anchor` path (lines 423-447) loads notebook papers as seeds and fetches candidates from OpenAlex. The `notebook_intent` type needs the SAME candidate-fetching behavior. Change the condition on line 423 from:
   ```r
   if (anchor_type == "notebook_anchor") {
   ```
   to:
   ```r
   if (anchor_type %in% c("notebook_anchor", "notebook_intent")) {
   ```
   This reuses the entire notebook-as-seeds + fetch-from-OpenAlex pipeline without duplication.

3. **Seed abstracts block** (lines 504-513): Currently for `notebook_anchor`, the code queries abstracts from the anchor notebook. The same behavior is needed for `notebook_intent`. Change line 504 from:
   ```r
   seed_abstracts <- if (anchor_type == "notebook_anchor") {
   ```
   to:
   ```r
   seed_abstracts <- if (anchor_type %in% c("notebook_anchor", "notebook_intent")) {
   ```

4. **Intent text block** (line 515): Currently intent is captured only for "intent" and "both". Add "notebook_intent" to the set. Change:
   ```r
   intent_text <- if (anchor_type %in% c("intent", "both")) input$anchor_intent else NULL
   ```
   to:
   ```r
   intent_text <- if (anchor_type %in% c("intent", "both", "notebook_intent")) input$anchor_intent else NULL
   ```
   This is the KEY change that makes "notebook_intent" different from plain "notebook_anchor" -- the intent text is combined with seed abstracts in `build_semantic_query()` to produce a richer semantic query.

5. **Source notebook ID for ragnar path** (line 520-521): Currently `notebook_anchor` sets `source_nb_id` to NULL (candidates are fetched, not from an existing notebook). The same applies to `notebook_intent`. Change:
   ```r
   source_nb_id <- if (anchor_type == "notebook_anchor") NULL
   ```
   to:
   ```r
   source_nb_id <- if (anchor_type %in% c("notebook_anchor", "notebook_intent")) NULL
   ```

6. **Effective source metadata** (lines 597-604): For DB storage, `notebook_intent` should behave like `notebook_anchor` -- source_type is "fetch" and the anchor notebook ID is stored. Change:
   ```r
   effective_source_type <- if (anchor_type == "notebook_anchor") "fetch" else input$source_type
   effective_source_nb <- if (anchor_type == "notebook_anchor") {
   ```
   to:
   ```r
   effective_source_type <- if (anchor_type %in% c("notebook_anchor", "notebook_intent")) "fetch" else input$source_type
   effective_source_nb <- if (anchor_type %in% c("notebook_anchor", "notebook_intent")) {
   ```

7. **Anchor intent DB save** (line 610): Currently the intent is saved for "intent" and "both". Add "notebook_intent". Change:
   ```r
   anchor_intent = if (anchor_type %in% c("intent", "both")) input$anchor_intent else NULL,
   ```
   to:
   ```r
   anchor_intent = if (anchor_type %in% c("intent", "both", "notebook_intent")) input$anchor_intent else NULL,
   ```

**Summary of the pattern:** Every place that checks `anchor_type == "notebook_anchor"` should also match `"notebook_intent"`. Every place that checks for intent types (`"intent"`, `"both"`) should also match `"notebook_intent"`. The `%in%` operator makes this clean.

**Do NOT modify research_refiner.R** -- the business logic functions (`fetch_candidates_from_seeds`, `build_semantic_query`, `score_with_temp_ragnar`, etc.) already accept seeds and intent as separate parameters and compose them correctly. The change is purely in how the Shiny module wires inputs to those functions.
