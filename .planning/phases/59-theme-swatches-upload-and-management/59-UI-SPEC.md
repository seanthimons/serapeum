## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Gap between swatch dot and next dot |
| sm | 8px | Padding inside swatch dot cluster; gap between × button and theme name |
| md | 16px | Default input spacing inside modal sections; fileInput margin |
| lg | 24px | Section gap between "Theme" row and upload link row |
| xl | 32px | Modal section separation (between Options h6 and its inputs) |
| 2xl | 48px | Not used in this phase |
| 3xl | 64px | Not used in this phase |

Exceptions: Swatch dots are 10px diameter (not a spacing token — a fixed size for color preview circles). The × delete button touch/click target is padded to 24px × 24px minimum so it is reliably clickable without triggering theme selection.

**Source:** Existing modal uses Bootstrap utility classes (`mb-3`, `mb-4`, `p-3`). 8-point scale maps to Bootstrap 5 spacing units.

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body / input label | 16px (1rem) | 400 (regular) | 1.5 |
| Compact label / helper text | 14px (0.875rem) | 400 (regular) | 1.4 |
| Section heading (h6) | 14px (0.875rem) | 600 (semibold) | 1.2 |
| Upload link / secondary action | 14px (0.875rem) | 400 (regular) | 1.4 |

**Rules:**
- Section headings inside the modal use `h6` with `class = "fw-semibold"` — this is the existing pattern from `mod_slides_modal_ui()` line 21.
- The upload trigger renders as a small link-style button (`actionLink` or `tags$button` styled as anchor), 14px, weight 400.
- Theme name text inside the dropdown option renders at 14px (browser default for `<option>` elements).
- Swatch dots render at 10px diameter with 0 text — purely visual.

**Source:** Observed from existing `mod_slides_modal_ui()` and Bootstrap 5 base typography.

## Color

| Role | Value (Light / Dark) | Usage |
|------|----------------------|-------|
| Dominant (60%) | Latte base `#eff1f5` / Mocha base `#1e1e2e` | Modal dialog background, body surface |
| Secondary (30%) | Latte mantle `#e6e9ef` / Mocha surface0 `#313244` | Modal header, bordered document list, selectInput backgrounds |
| Accent (10%) | Latte lavender `#7287fd` / Mocha lavender `#b4befe` | Upload button border/text (link style); active theme row highlight |
| Destructive | Latte red `#d20f39` / Mocha red `#f38ba8` | × delete icon color only |

Accent reserved for:
1. The "Upload custom theme (.scss)" link/button text and underline
2. The active/selected item highlight in the theme dropdown (Bootstrap default `option:checked` highlight, already inherits `primary`)

Destructive (`danger` red) reserved for:
1. The × delete icon rendered next to each custom theme in the dropdown

**Swatch dot colors:** The three color dots (bg/fg/accent) for each theme render the theme's own hex values — they are not part of the app's own color contract. Use `border-radius: 50%` circles sized 10px × 10px, `display: inline-block`, with 2px `border` in `LATTE$surface2` (`#acb0be`) / `MOCHA$surface1` (`#45475a`) so white background swatches are visible.

**Source:** `R/theme_catppuccin.R` LATTE/MOCHA constants; `app.R` `bs_theme()` mapping; semantic color policy in `theme_catppuccin.R` lines 86–120.

## Component Inventory

### 1. Theme Dropdown (replaces existing `selectInput`)

**Type:** `selectizeInput` with custom HTML rendering via `render` option, or `selectInput` with `choices` built from `HTML()` labels.

**Structure:**
```
[Option group: Built-in]
  [● ● ●] default
  [● ● ●] beige
  ... (11 built-in themes)

[Option group: Custom]
  [● ● ●] epa-owm  [×]
  [● ● ●] my-theme [×]
```

**Behavior:**
- Groups rendered via `optgroup` labels: "Built-in" and "Custom"
- Built-in entries: three colored dots + theme name. No delete control.
- Custom entries: three colored dots + theme name + × button
- × click: stops event propagation (does not change selected theme), triggers server-side delete
- If no custom themes exist: "Custom" group is absent from the dropdown
- Selected state: lavender highlight (Bootstrap `primary` color — inherited from `bs_theme`)

**Implementation note (Claude's Discretion):** Use `selectizeInput` with `options = list(render = I(...))` and a JavaScript `render.option` function to inject swatch HTML. The × button uses `data-theme` attribute and a Shiny custom message handler to avoid full re-render on delete.

---

### 2. Upload Link

**Placement:** Directly below the theme dropdown, inside the existing Options section `div(class = "mb-3", ...)`.

**Rendered as:** `actionLink(ns("upload_theme_link"), "Upload custom theme (.scss)", icon = icon("upload"))` styled as small muted link (Bootstrap `text-muted` + `small` class), or `tags$label` triggering a hidden `fileInput`.

**Copy:** "Upload custom theme (.scss)" — 14px, weight 400, lavender color on hover.

---

### 3. File Input (hidden)

**Type:** `fileInput(ns("theme_file"), NULL, accept = ".scss")` — rendered off-screen or as `display: none`, triggered by the upload link click via JavaScript.

**Constraint:** `accept = ".scss"` restricts file picker OS dialog to `.scss` files only (not enforced server-side alone).

---

### 4. Validation Error Display

**Placement:** Inline, between the upload link and the theme dropdown (or directly below upload link).

**Rendered as:** `uiOutput(ns("upload_error"))` — renders a `div` with `class = "alert alert-danger py-1 px-2 small mt-1"` when validation fails.

**Disappears:** On next successful upload or when the user opens a fresh modal.

---

### 5. Upload Success Notification

**Type:** `showNotification("Theme uploaded.", type = "message", duration = 3)` — uses existing Shiny notification pattern from the slides module.

**Copy:** "Theme '[filename]' uploaded." — no icon needed beyond the system notification styling.

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA (upload trigger) | "Upload custom theme (.scss)" |
| Upload success notification | "Theme '[filename]' uploaded." |
| Empty state (no custom themes) | Custom group absent — no empty state message needed in dropdown |
| Validation error — missing markers | "Invalid .scss file. Must contain `/*-- scss:defaults --*/` and `/*-- scss:rules --*/` sections." |
| Validation error — wrong file type | "Please upload a .scss file." |
| Delete (no confirmation) | No copy — × icon only. Deletion is immediate. |
| Theme dropdown label | "Theme" (unchanged from current `selectInput` label) |
| Built-in group label | "Built-in" |
| Custom group label | "Custom" |

**Copy rules:**
- Validation errors are inline (alert), not modal — they appear near the upload link, not in a separate dialog.
- No confirmation dialog for theme deletion — the × button is the only affordance (locked decision from CONTEXT.md).
- Duplicate filename upload: silent overwrite, same success notification as first upload.

**Source:** Locked in 59-CONTEXT.md `## Implementation Decisions` (upload validation, no confirmation for delete, silent overwrite).

## Interaction States

### Theme Dropdown

| State | Visual |
|-------|--------|
| Default | Standard selectInput appearance; swatch dots visible for all entries |
| Hover (option) | Bootstrap default highlight (lavender background) |
| Selected | Selected theme name shown; swatch dots visible in selected display area |
| × hover | × icon color shifts from `text-muted` gray to danger red (`#d20f39` / `#f38ba8`) |
| × click | Theme removed from list; dropdown collapses to next available theme or built-in default |
| No custom themes | Custom optgroup absent entirely |

### Upload Link

| State | Visual |
|-------|--------|
| Default | Muted small link, 14px, `text-muted` gray |
| Hover | Lavender color (`#7287fd` / `#b4befe`), underline |
| Loading (file picker active) | No visual change — OS file picker is open |
| Uploading/processing | Brief — `showNotification("Uploading...")` with no duration, removed on completion |

### Validation Error

| State | Visual |
|-------|--------|
| No error | `uiOutput` renders nothing |
| Error present | Red alert (`alert-danger`) inline below upload link, 14px text |
| Error cleared | Cleared on next upload attempt |
