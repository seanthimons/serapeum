  <files>R/themes.R, tests/testthat/test-themes.R</files>
  <behavior>
    - BUILTIN_THEME_SWATCHES is a named list of 11 entries (default, beige, blood, dark, league, moon, night, serif, simple, sky, solarized) each with bg, fg, accent character strings
    - parse_scss_swatches("") returns fallback list(bg="#FFFFFF", fg="#000000", accent="#157efb")
    - parse_scss_swatches with epa-owm.scss content returns bg="#FFFFFF", fg="#212529", accent="#0D5C63" (resolving $epa-white -> #FFFFFF, $epa-text -> #212529, $epa-blue -> #0D5C63)
    - parse_scss_swatches with direct hex values ($body-bg: #FF0000;) returns the hex directly
    - validate_scss_file returns TRUE for text containing both section markers, FALSE otherwise
    - list_custom_themes returns empty list for non-existent directory
    - list_custom_themes returns list of theme metadata (filename, label, bg, fg, accent) for directory with .scss files
    - build_theme_choices_df with no custom themes returns 11-row data.frame with columns: value, label, bg, fg, accent, group (all "builtin")
    - build_theme_choices_df with custom themes appends rows with group="custom" and value=filename (not full path)
    - After file.remove on a theme, list_custom_themes no longer includes it
  </behavior>
  <implementation>
    Create R/themes.R with the following exports:

    1. BUILTIN_THEME_SWATCHES — named list mapping each of 11 RevealJS theme names to list(bg, fg, accent) with approximate hex values:
       - default: bg="#FFFFFF", fg="#000000", accent="#157efb"
       - beige: bg="#F7F3DE", fg="#333333", accent="#8B743D"
       - blood: bg="#160a0a", fg="#EEEEE2", accent="#AA2233"
       - dark: bg="#111111", fg="#EEEEE2", accent="#E7AD52"
       - league: bg="#1C1E20", fg="#EEEEE2", accent="#F0DB4F"
       - moon: bg="#002b36", fg="#93a1a1", accent="#268bd2"
       - night: bg="#1C1E20", fg="#EEEEE2", accent="#F28705"
       - serif: bg="#F0F1EB", fg="#333333", accent="#51483D"
       - simple: bg="#FFFFFF", fg="#000000", accent="#007CAD"
       - sky: bg="#F2F6F9", fg="#333333", accent="#3B759E"
       - solarized: bg="#FDF6E3", fg="#333333", accent="#259286"

    2. parse_scss_swatches(scss_text) — Extracts bg/fg/accent hex colors from the scss:defaults block:
       a. Extract the text between `/*-- scss:defaults --*/` and the next `/*--` or end of file
       b. Build a variable resolution table: scan for all `$varname: #hexval;` patterns and store in a named list
       c. Look for target variables in priority order:
          - bg: $body-bg, $backgroundColor
          - fg: $body-color, $mainColor
          - accent: $link-color, $linkColor, $presentation-heading-color
       d. For each target variable found, check if its value is a hex literal (#XXXXXX) or a variable reference ($varname). If variable reference, look up in the resolution table. Recurse one level (sufficient for real themes).
       e. Return list(bg=, fg=, accent=) with defaults "#FFFFFF"/"#000000"/"#157efb" for missing values

    3. validate_scss_file(scss_text) — Returns TRUE if scss_text contains both `/*-- scss:defaults --*/` AND `/*-- scss:rules --*/`, FALSE otherwise

    4. list_custom_themes(themes_dir = "data/themes") — Returns a list of lists, each with: filename (basename), label (file_path_sans_ext of basename), bg, fg, accent. Returns empty list() if directory does not exist or contains no .scss files. Calls parse_scss_swatches on each file.

    5. build_theme_choices_df(custom_themes = list()) — Returns a data.frame with columns: value, label, bg, fg, accent, group. Built-in rows have group="builtin", value=theme name (e.g. "default"), label=Title Case name. Custom rows have group="custom", value=filename (e.g. "epa-owm.scss"), label=file_path_sans_ext (e.g. "epa-owm"). The custom value is the filename, NOT the full path — the server wiring in Plan 02 will prepend "data/themes/" when setting custom_scss.
  </implementation>
