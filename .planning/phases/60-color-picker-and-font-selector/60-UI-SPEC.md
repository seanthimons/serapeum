## Spacing Scale

Declared values (multiples of 4 only). Bootstrap utility classes map directly to these tokens.

| Token | Value | Bootstrap Class | Usage |
|-------|-------|-----------------|-------|
| xs | 4px | `gap-1`, `p-1` | Icon-to-text gap, swatch dot spacing |
| sm | 8px | `gap-2`, `mb-2` | Color picker label margin, inline element gaps |
| md | 16px | `p-3`, `mb-3` | Panel internal padding, element spacing |
| lg | 24px | `mb-4` | Section separation within modal |
| xl | 32px | `p-4` | Not used in this phase |
| 2xl | 48px | — | Not used in this phase |
| 3xl | 64px | — | Not used in this phase |

Exceptions:
- Native `<input type="color">` swatch: fixed 40px wide × 38px tall (aligns with Bootstrap form control height)
- Hex text field: fixed 100px wide (7-character hex + `#` fits comfortably)
- Swatch dots in theme dropdown: 10px diameter (existing Phase 59 established value — do not change)
- Touch target minimum: 44px height for the color swatch + hex pair row (accessibility floor)

## Color

The Catppuccin semantic color policy (`R/theme_catppuccin.R` lines 79–120) governs all color usage in this phase. New UI must not introduce any colors outside this policy.

### App UI Colors (the picker panel's own chrome)

| Role | Light (Latte) | Dark (Mocha) | Usage |
|------|---------------|--------------|-------|
| Dominant (60%) | `#eff1f5` (LATTE$base) | `#1e1e2e` (MOCHA$base) | Modal background surface |
| Secondary (30%) | `#e6e9ef` (LATTE$mantle) | `#181825` (MOCHA$mantle) | Collapsible panel background (`bg-body-secondary` Bootstrap class) |
| Accent (10%) | `#7287fd` (LATTE$lavender) | `#b4befe` (MOCHA$lavender) | "Save as custom theme" primary action button only (`.btn-primary`) |
| Destructive | `#d20f39` (LATTE$red) | `#f38ba8` (MOCHA$red) | Not applicable in this phase — no delete actions in the picker panel |

**Accent reserved for:** The "Save as custom theme" button exclusively. The panel toggle link/button uses `.btn-link` or plain text anchor — not accent color.

**Validation state color:**
- Hex field invalid (non-`#RRGGBB` on blur): `border-color: #d20f39` (LATTE$red) / `#f38ba8` (MOCHA$red) — use Bootstrap `.is-invalid` class which inherits `--bs-danger`
- Hex field valid/neutral: default Bootstrap form control border

### Slide Theme Colors (what the pickers control — not app UI)

The 4 color pickers let users choose arbitrary hex colors for their slides. These are pass-through values — no app-level color policy applies to the picker values themselves.

## Component Inventory

These are the specific UI elements the executor must build for Phase 60.

### 1. Collapsible Panel Toggle

```
[chevron-right icon] Customize colors & font
```

- Element: Bootstrap collapse trigger (`data-bs-toggle="collapse"`) or `shinyjs::toggle()` on click
- Placement: Full-width div directly below the theme selectizeInput + upload link block (after line 145 in mod_slides.R), before the speaker notes checkbox
- Default state: **collapsed**
- Toggle icon: `icon("chevron-right")` rotates 90° when expanded (CSS `transition: transform 0.2s`)
- Label style: `class = "small text-muted d-flex align-items-center gap-1"` — matches the upload link visual weight
- Source: CONTEXT.md — "Collapsible panel below the existing theme dropdown — collapsed by default"

### 2. Color Picker Pair (×4 instances)

Each of BG, Text, Accent, Link is one pair:

```
[Label: 12px semibold]
[color swatch 40×38px] [hex text field 100px wide]
```

- Native `<input type="color">` for the swatch — `style = "width:40px;height:38px;padding:2px;border:1px solid #ced4da;border-radius:4px;cursor:pointer;"`
- Shiny `textInput()` for hex field — `width = "100px"`, `placeholder = "#RRGGBB"`
- JS sync (swatch → hex): `oninput` listener on color input updates text field value (uppercase), dispatches `change` event
- JS sync (hex → swatch): `oninput` listener on text field sets color input value when value matches `^#[0-9A-Fa-f]{6}$`
- Validation: on `blur` of hex field, if value does not match `^#[0-9A-Fa-f]{6}$`, add Bootstrap class `is-invalid` to the text field; remove on valid input
- Row layout: `d-flex align-items-center gap-2`
- Input IDs (namespaced): `bg_hex`, `text_hex`, `accent_hex`, `link_hex` (Shiny-registered); `bg_swatch`, `text_swatch`, `accent_swatch`, `link_swatch` (DOM-only, not Shiny inputs)
- Source: CONTEXT.md — "native HTML color input (clickable swatch) paired with an editable hex text field"

### 3. 2×2 Color Grid Layout

```
[ BG picker pair  ]  [ Text picker pair  ]
[ Accent pair     ]  [ Link picker pair  ]
```

- `layout_columns(col_widths = c(6, 6), ...)` — two-column, two-row
- Top row: BG (left), Text (right)
- Bottom row: Accent (left), Link (right)
- Source: CONTEXT.md — "2×2 grid layout: BG/Text on top row, Accent/Link on bottom row"

### 4. Font Selector

```
Font [grouped selectInput]
```

- `selectInput(ns("font"), "Font", choices = list(...), selected = "Source Sans Pro")`
- Grouped choices (plain text, no font preview rendering):

```r
list(
  "Sans-serif" = c(
    "Source Sans Pro" = "Source Sans Pro",
    "Lato"           = "Lato",
    "Fira Sans"      = "Fira Sans",
    "Roboto"         = "Roboto",
    "Open Sans"      = "Open Sans"
  ),
  "Serif" = c(
    "Merriweather"    = "Merriweather",
    "PT Serif"        = "PT Serif",
    "Roboto Slab"     = "Roboto Slab",
    "Playfair Display"= "Playfair Display"
  ),
  "Monospace" = c(
    "IBM Plex Mono" = "IBM Plex Mono",
    "Fira Code"     = "Fira Code"
  )

```

- Total: 11 fonts across 3 categories (satisfies THME-11 ≥ 10 fonts)
- Default: `"Source Sans Pro"`
- Input ID: `font`
- Source: CONTEXT.md — "~10-12 curated, widely-available professional fonts grouped by category"

### 5. Save Row

```
[text field: theme name placeholder] [Save as custom theme btn]
```

- `textInput(ns("custom_theme_name"), NULL, placeholder = "Theme name...")` — inline, no label, flexible width
- `actionButton(ns("save_custom_theme"), "Save as custom theme", class = "btn-primary btn-sm", icon = icon_save())`
- Layout: `d-flex align-items-center gap-2 mt-3`
- Source: CONTEXT.md — "'Save as custom theme' button with inline text input for theme name"

### 6. Live Swatch Dot Update

When any hex picker value changes (on blur, not on every keypress):
- Rebuild choices data frame for `updateSelectizeInput` with overridden `bg`/`fg`/`accent` values for the currently selected theme row
- This refreshes the three swatch dots in the dropdown without closing it
- Implementation: `observeEvent` on `c(input$bg_hex, input$text_hex, input$accent_hex)` with debouncing — update on blur via `shinyjs` or by triggering update on the change event (not oninput)
- Source: CONTEXT.md — "Live swatch update: as user changes picker values, the 3 swatch dots in the theme dropdown update"

### 7. Panel Pre-fill Behavior

On theme dropdown change (`observeEvent(input$theme)`):
- Built-in themes: fill from `BUILTIN_THEME_SWATCHES[[theme]]` — link defaults to accent value, font defaults to "Source Sans Pro"
- Custom themes: parse via `parse_scss_colors_full(path)` — extracts all 4 colors + `$mainFont`
- If panel is collapsed while theme changes: still update inputs (hidden inputs retain value for when panel opens)
- Source: CONTEXT.md — "Expanding the panel pre-fills pickers with the currently selected theme's colors"

## Interaction States

| State | Trigger | Visual Response |
|-------|---------|-----------------|
| Panel collapsed (default) | Page load / after save | Chevron points right; panel height = 0 (Bootstrap collapse) |
| Panel expanding | Click toggle | Chevron rotates 90° (CSS transition 0.2s); panel slides open |
| Color swatch clicked | User clicks native color input | OS color picker opens |
| Swatch changes | User picks color in OS picker | Paired hex field updates in real time (oninput) |
| Hex field typed | User types in text field | Swatch color updates when value is valid hex; no update on partial input |
| Hex field blur — invalid | Value is not `#RRGGBB` | `.is-invalid` border applied (Bootstrap red) |
| Hex field blur — valid | Value matches `^#[0-9A-Fa-f]{6}$` | `.is-invalid` removed; dropdown swatch dots update |
| Theme dropdown changes | User selects different theme | All 4 hex fields + font selector update to new theme's values |
| Save button clicked — name empty | `custom_theme_name` is blank | Button press silently blocked (`req()`); focus moves to name field |
| Save button clicked — name valid | Theme name present, all hex values valid | .scss written, dropdown refreshed, new theme auto-selected, panel collapses, toast shown |
| Phase 61 populates pickers | AI calls `updateTextInput` / `updateSelectInput` | Same visual response as if user typed — hex fields update, swatch dots update, font selector updates |

## Generated .scss Contract

The executor must produce exactly this structure from `generate_custom_scss()`:

```scss
/*-- scss:defaults --*/

$backgroundColor: #FFFFFF;
$mainColor: #000000;
$linkColor: #157efb;
$accentColor: #157efb;
$mainFont: "Source Sans Pro", sans-serif;

/*-- scss:rules --*/

/* Generated by Serapeum theme customizer */
.reveal h1, .reveal h2, .reveal h3 {
  color: $accentColor;
}
```

Rules:
- Font name always double-quoted in `$mainFont` value (prevents SCSS parse errors on multi-word names)
- Fallback stack appended: `"Font Name", sans-serif`
- `$accentColor` wired to heading colors in `scss:rules` block (makes accent picker meaningful — Claude's Discretion resolution)
- Filename: `gsub("[^a-zA-Z0-9_-]", "-", name)` + `.scss`
- Write path: `data/themes/{sanitized-name}.scss`
- Duplicate names: overwrite silently (consistent with Phase 59 upload behavior)
