## Color

Catppuccin palette. Both light and dark modes must be respected — all new elements must use Bootstrap semantic variables, not hardcoded hex, unless within a `[data-bs-theme="dark"]` override block in custom.css.

| Role | Light (LATTE) | Dark (MOCHA) | Usage |
|------|---------------|--------------|-------|
| Dominant (60%) | #eff1f5 (base) | #1e1e2e (base) | Modal background, page surfaces |
| Secondary (30%) | #e6e9ef (mantle) / #ccd0da (surface0) | #313244 (surface0) | Customize panel `bg-body-secondary`, card backgrounds |
| Accent (10%) | #7287fd (lavender, mapped to `primary`) | #b4befe (lavender) | AI Generate button (btn-primary), Generate button in popover, spinner color |
| Destructive | #d20f39 (red, mapped to `danger`) | #f38ba8 (red) | Not used in this phase — validation error toasts use `type = "error"` which maps to danger automatically |

Accent reserved for:
- The "AI Generate" button (btn-primary, next to Upload link — peer status, same primary class)
- The "Generate" button inside the popover (btn-primary)
- The spinner during LLM call (`spinner-border text-primary`)
- The "Regenerate" button inside the customize panel (btn-outline-secondary — same outline style as existing heal chips, NOT btn-primary, to signal secondary action)

Color notes:
- Upload link is `.small.text-muted` — the AI Generate button should match as a peer: use `btn-link btn-sm` or `small text-muted` inline style to stay visually subordinate to the theme dropdown
- Error toasts call `showNotification(..., type = "error")` — Bootstrap maps `danger` automatically
- Warning toasts for font fallback call `showNotification(..., type = "warning")`
- No new hardcoded hex values. Use Bootstrap semantic classes (`text-muted`, `btn-primary`, `bg-body-secondary`, `border`, `rounded`) throughout

## Component Inventory

New components introduced in this phase (referenced by executor):

### 1. AI Generate Trigger — inline link next to Upload

- Element: `tags$a()` or `actionLink()` styled to match the existing Upload label
- Class: `small text-muted d-inline-flex align-items-center gap-1`
- Style: `cursor:pointer; margin-bottom:0;` (matches Upload link at line 191-194)
- Icon: `icon("wand-magic-sparkles")` (FontAwesome solid, same family as `icon_wand()` used in Generate footer button)
- Label: " AI Generate"
- Placement: Below the theme selectizeInput, on the same line as the Upload link — use `d-flex gap-3` wrapper to align Upload and AI Generate side-by-side

### 2. AI Theme Popover

- Trigger: Bootstrap 5 popover on the AI Generate link
- Popover content: rendered via `tags$div()` in server with `insertUI` or as a static Shiny popover wrapper
- Implementation: Since Shiny modals can't easily nest another modal, use a `shinyBS::bsPopover` or a manual Bootstrap 5 `data-bs-toggle="popover"` with `data-bs-html="true"` and HTML content injected via JS
- Preferred pattern: `actionButton` with `data-bs-toggle="popover"` + `data-bs-html="true"` + `data-bs-content` set to the textarea HTML, matching the popover pattern used elsewhere in the app
- Popover width: 320px (set via `.popover { max-width: 320px; }` CSS override — default is 276px, too narrow for textarea)
- Popover placement: `bottom` (theme area is mid-modal; bottom avoids clipping)

### 3. Description Textarea (inside popover)

- Element: `textarea` with 2-3 rows
- Placeholder: `e.g., ocean blues, dark background, modern sans-serif font`
- Class: `form-control form-control-sm`
- Width: 100% of popover content width
- No character limit enforced in UI (LLM prompt handles context length)

### 4. Generate Button (inside popover)

- Element: `actionButton()` or `tags$button()`
- Label: "Generate theme"
- Class: `btn btn-primary btn-sm mt-2 w-100`
- Spinner state: `disabled` attribute + inner HTML replaced with `<span class="spinner-border spinner-border-sm" role="status"></span> Generating...` during LLM call
- Pattern: Mirrors chat send button spinner from v5.0

### 5. Regenerate Button (inside customize panel)

- Element: `actionButton(ns("regenerate_theme"), ...)`
- Label: "Regenerate"
- Class: `btn btn-outline-secondary btn-sm`
- Icon: `icon("rotate")` (FontAwesome)
- Placement: Inside the customize panel `div`, below the font selector and above the save row — rendered only after a successful AI generation (conditional `uiOutput`)
- Spinner state: Same disabled + spinner pattern as Generate button

## Interaction States

### AI Generate Button

| State | Visual |
|-------|--------|
| Default | `small text-muted` inline link with wand icon — subordinate to dropdown |
| Hover | Browser default link hover (cursor pointer, underline or color shift via Bootstrap) |
| Popover open | Popover visible below button |

### Generate Button (popover)

| State | Visual |
|-------|--------|
| Default | `btn-primary btn-sm w-100` — full popover width, accent color |
| Disabled (no text entered) | Not enforced — allow empty submit; LLM will return an error handled by retry logic |
| Loading | Disabled + `spinner-border-sm` + "Generating..." label |
| Success | Popover closes; customize panel auto-expands (server-driven via `session$sendCustomMessage("collapse_panel", ...)` in reverse — i.e. expand via `bootstrap.Collapse.getOrCreateInstance(el).show()`) |
| Error (toast shown) | Popover remains open; button re-enables; user sees `showNotification` error toast |

### Customize Panel (post-AI-generation)

| State | Visual |
|-------|--------|
| Before AI generation | Collapsed (existing Phase 60 behavior) |
| After successful generation | Auto-expanded via JS message; color pickers populated with AI values; Regenerate button visible |
| After manual tweaking | Color pickers reflect user edits; Regenerate button remains visible until save |
| After save | Panel collapses (existing Phase 60 behavior); Regenerate button disappears |

### Regenerate Button

| State | Visual |
|-------|--------|
| Default | `btn-outline-secondary btn-sm` with rotate icon |
| Loading | Disabled + spinner (same pattern as Generate button) |
| Success | Color pickers update in-place with new AI values |
| Error | `showNotification` error toast; button re-enables |

## Accessibility

- The AI Generate trigger must have `aria-label="Generate theme with AI"` since the link text "AI Generate" is terse
- The popover trigger button must have `aria-haspopup="dialog"` and `aria-expanded` toggled by Bootstrap 5
- The Generate button inside the popover must have `aria-busy="true"` set during loading state
- Spinner elements must include `<span class="visually-hidden">Loading</span>` inside the spinner span
- The Regenerate button must have `aria-label="Regenerate theme with AI"` to distinguish from other generate buttons on the page
- All form labels in the popover must be associated to inputs via `for` / `id` pairing (Bootstrap `form-label` + `form-control` pattern)
- Color picker pair hex inputs already have `is-invalid` class feedback from Phase 60 — no changes needed

## Implementation Constraints

1. **No `useShinyjs()`** — app.R does not call `useShinyjs()`. Use `session$sendCustomMessage()` for all JS-driven panel state changes (established in Phase 60 — STATE.md decision log).
2. **No `display:none` for file inputs** — not relevant to this phase but noted for context.
3. **Bootstrap 5 Collapse API for expand** — use `bootstrap.Collapse.getOrCreateInstance(el).show()` to programmatically expand the customize panel after AI generation (mirror of Phase 60's collapse handler).
4. **Popover HTML injection** — if using `data-bs-content` with HTML, set `data-bs-html="true"` on the trigger element. Shiny input events inside popover HTML need explicit `Shiny.setInputValue()` calls since Bootstrap popovers are not Shiny-aware.
5. **Preferred popover approach** — consider rendering the "AI Generate" area as a small inline form that expands/collapses (Bootstrap collapse) rather than a true BS5 popover, to avoid Shiny input registration issues with dynamically inserted DOM. The design contract requires the collapsed/expanded feel — the exact mechanism (popover vs. collapse block) is executor discretion as long as it matches the visual spec.
6. **DOMContentLoaded guard** — any JS listeners on modal-rendered elements must use the `DOMContentLoaded` guard pattern (Phase 60 decision, mod_slides.R lines 31-66).
