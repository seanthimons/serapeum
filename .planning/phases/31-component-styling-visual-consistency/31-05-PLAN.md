  <action>
**Root cause:** Custom JavaScript toggle (app.R lines 54-66) modifies `data-bs-theme` attribute client-side without notifying R server. `thematic::thematic_shiny()` relies on `getCurrentOutputInfo()` reactive values (bg/fg) that require server-side theme changes via `session$setCurrentTheme()` or `bslib::input_dark_mode()`.

**Solution:** Replace custom button with `bslib::input_dark_mode()` widget, which integrates with Shiny's reactive system and enables thematic to detect theme changes.

**Step 1 - Remove custom JS toggle from UI:**

In `app.R`, delete lines 54-66 (the entire custom dark mode toggle button):

```r
    tags$button(
      id = "dark_mode_toggle",
      class = "btn btn-sm btn-outline-secondary border-0",
      onclick = "
        const html = document.documentElement;
        const current = html.getAttribute('data-bs-theme');
        const next = current === 'dark' ? 'light' : 'dark';
        html.setAttribute('data-bs-theme', next);
        localStorage.setItem('theme', next);
        this.innerHTML = next === 'dark' ? '<i class=\"fa fa-sun\"></i>' : '<i class=\"fa fa-moon\"></i>';
      ",
      icon("moon")
    )
```

**Step 2 - Add input_dark_mode() to navbar:**

Replace the deleted button with `bslib::input_dark_mode()` at the same location (after the title div, before closing the title argument):

```r
    bslib::input_dark_mode(id = "dark_mode")
```

The complete title argument should now look like:

```r
  title = div(
    class = "d-flex align-items-center gap-2 w-100",
    div(
      class = "d-flex align-items-center gap-2",
      icon("book-open"),
      "Serapeum"
    ),
    bslib::input_dark_mode(id = "dark_mode")
  ),
```

**Step 3 - Add localStorage persistence in server:**

In the server function (after `thematic::thematic_shiny()` call around line 234), add an observer to persist theme preference:

```r
  # Persist theme preference to localStorage
  observeEvent(input$dark_mode, {
    session$sendCustomMessage(
      type = "set-theme-storage",
      message = list(theme = if(input$dark_mode) "dark" else "light")
    )
  }, ignoreNULL = FALSE, ignoreInit = TRUE)
```

**Step 4 - Add JavaScript handler for localStorage:**

In the UI, after the theme definition (after line 100, before the `page_navbar()` call), add a script tag to handle localStorage:

```r
  tags$script(HTML("
    Shiny.addCustomMessageHandler('set-theme-storage', function(message) {
      localStorage.setItem('theme', message.theme);
    });

    // Restore theme on page load
    document.addEventListener('DOMContentLoaded', function() {
      const savedTheme = localStorage.getItem('theme');
      if (savedTheme) {
        Shiny.setInputValue('dark_mode', savedTheme === 'dark', {priority: 'event'});
      }
    });
  ")),
```

**Why this works:** `input_dark_mode()` calls `session$setCurrentTheme()` internally, which updates the reactive values that `getCurrentOutputInfo()` reads. When user toggles theme, thematic detects the change and re-renders plots with appropriate backgrounds.

**Note:** The `input_dark_mode()` widget automatically styles itself and positions in the navbar. It renders as a moon/sun icon similar to the custom button but with full Shiny integration.
