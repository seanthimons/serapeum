# Phase 52.1 Plan 01: Fix Search Refresh and Load More Bugs

**One-liner:** Consolidated duplicate search functions into unified do_search(mode), fixing triple-toast error cascade and NULL has_more silent failure with proper error short-circuiting and isTRUE() guard.

## Overview

Fixed two UAT-blocking bugs discovered in the search notebook:

1. **BUG-01 (Triple Toast):** Refresh button showed three toasts on API error - the error toast, plus "No papers found", plus "No new papers found". Caused by error handler returning `list(papers = list())` which allowed execution to continue downstream.

2. **BUG-02 (Silent Load More Failure):** Load More button silently did nothing when `pagination_state$has_more` was NULL (vs FALSE). Used bare `req(pagination_state$has_more)` which treats NULL and FALSE identically, providing no user feedback.

## What Was Built

### Unified do_search(mode) Function

Replaced two separate functions (`do_search_refresh()` and `do_load_more()`, ~290 lines total) with a single `do_search(mode)` function (~113 lines). Eliminates ~107 lines of duplicate code.

**Structure:**
1. `match.arg(mode)` for type-safe mode parameter validation
2. Common preamble (notebook validation)
3. Mode-specific guards (refresh: query check; load_more: NULL-safe has_more + spinner swap)
4. Common config and cursor setup
5. Single tryCatch wrapping entire withProgress block
6. Common paper processing (API call, pagination update, dedup, DB save)
7. Mode-specific success toasts

### Bug Fixes

**BUG-01 Fix:**
```r
# OLD (in do_search_refresh):
result <- tryCatch({
  search_papers(...)
}, error = function(e) {
  show_error_toast(...)
  return(list(papers = list()))  # ← Allows downstream execution!
})

# NEW (in do_search):
tryCatch({
  withProgress({
    result <- search_papers(...)
    # ... process papers ...
  })
}, error = function(e) {
  show_error_toast(...)
  return()  # ← Exits entire function, no downstream toasts
})
```

**BUG-02 Fix:**
```r
# OLD (in do_load_more):
req(pagination_state$has_more)  # ← Silent exit on NULL

# NEW (in do_search):
if (!isTRUE(pagination_state$has_more)) {
  showNotification("No more results available", type = "message")
  return()
}
```

## Key Decisions

1. **isTRUE() for NULL safety:** Using `isTRUE(pagination_state$has_more)` instead of bare `req()` provides explicit NULL handling and user feedback. `isTRUE(NULL)` returns FALSE (vs `req(NULL)` which silently halts reactive chain).

2. **Early return in error handler:** Error handler calls `return()` to exit the entire function, preventing any downstream showNotification calls. This is the root fix for the triple-toast cascade.

3. **match.arg() for mode validation:** Type-safe parameter validation at function entry. Invalid mode values throw immediate error instead of failing later.

4. **on.exit() for guaranteed cleanup:** Load More mode uses `on.exit()` to restore spinner icon and processing state, ensuring cleanup even if error occurs mid-execution.
