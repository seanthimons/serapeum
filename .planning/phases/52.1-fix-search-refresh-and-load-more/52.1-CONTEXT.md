**Gathered:** 2026-03-09

<domain>
## Phase Boundary

Fix two UAT-blocking issues from Phase 52: (1) Refresh triggers triple toast on error instead of one, (2) Load More silently does nothing. Consolidate the duplicate search execution code into a unified function.

</domain>

<decisions>
## Implementation Decisions

### Debug approach
- Start with code review to identify root causes (already done during discuss-phase — see root causes below)
- Add targeted instrumentation only if code review fix doesn't resolve at runtime
- Audit ALL if() conditions in both do_search_refresh and do_load_more for NULL safety

### Root causes identified
- **Refresh triple toast:** Inner tryCatch (line 2256) catches API errors but returns `list(papers = list(), ...)` — execution continues. `length(papers) == 0` triggers "No papers found" toast. After withProgress, `newly_added == 0` triggers "No new papers found" toast. Three independent code paths fire because the error handler returns a "success" value instead of stopping.
- **Load More silent failure:** `req(pagination_state$has_more)` at line 2380 silently exits if `has_more` is NULL (not FALSE). If pagination state wasn't properly initialized after the initial search, the req() swallows the click.

### Toast policy
- One toast per operation — error path shows only error toast, success path shows only success toast
- No cascading toasts — when an error occurs, downstream "no papers" toasts must NOT fire

### Fix scope
- Fix both Refresh AND Load More (same root cause family)
- Audit all if() conditions in both functions for potential NULL/NA inputs — prevent future whack-a-mole

### Error handling restructure
- Fix the control flow so errors short-circuit properly — Refresh error handler must return() from the entire function, not just from the tryCatch block
- Fix Load More's NULL vs FALSE guard on pagination_state$has_more

### Code consolidation
- Unify do_search_refresh and do_load_more into a single do_search() function with mode parameter (mode='refresh' or mode='load_more')
- Mode controls: cursor value (NULL for refresh, pagination_state$cursor for load_more), toast messages, UI state (spinner swap for load_more only)
- Single error handler, single paper-saving loop, single dedup path — eliminates ~80 lines of duplicate code and prevents future drift

### Toolbar button style (cosmetic) — DROPPED
- User decided cosmetic fixes are out of scope for this phase

</decisions>
