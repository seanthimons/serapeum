# Phase 53.1: Import Run Dialog Appearing at Bottom of Abstract Notebook - Research

**Researched:** 2026-03-11
**Domain:** R/Shiny UI module refactoring, Bootstrap 5 collapse component, modal dialog modification
**Confidence:** HIGH

## Summary

This phase fixes a UI bug where import run history cards render as visible, unstyled content at the bottom of the search notebook instead of being contained within the import modal. The root cause is `mod_bulk_import_ui()` outputting `uiOutput(ns("import_history"))` that renders directly into the page flow. The fix involves moving this history UI into the existing import modal as a collapsible section, adding module parameterization to control history visibility, and removing delete functionality.

The technical domain is well-understood: Bootstrap 5 collapse components, Shiny module parameterization, and modal dialog composition. All patterns already exist in the codebase (keyword/journal filter collapses, module optional parameters).

**Primary recommendation:** Use Bootstrap 5 collapse pattern (matching existing filter panels), parameterize the module with `show_history` flag, render history inside modal footer when enabled, and strip delete handlers entirely.

### Anti-Patterns to Avoid
- **Rendering uiOutput in module UI root:** Creates visible page content instead of modal-scoped UI
- **Leaving dead code paths:** When `show_history = FALSE`, don't render history or wire delete observers
- **Hard-coded history visibility:** Must be parameterized — sidebar import never shows history
- **Unconditional delete handlers:** Delete observers should only exist when `show_history = TRUE`

## Don't Hand-Roll

| Problem | Don't Build | Use Instead | Why |
|---------|-------------|-------------|-----|
| Collapsible sections | Custom JS toggle | Bootstrap 5 collapse (`data-bs-toggle="collapse"`) | Already loaded via bslib, keyboard accessible, theme-aware |
| Count badges | Custom styled span | Bootstrap badge (`class = "badge bg-secondary"`) | Consistent with existing search result type badges |
| Modal modification | New modal component | Extend existing `showModal()` footer | Maintains existing validation workflow and UI consistency |

**Key insight:** Bootstrap 5 ships with bslib and provides collapse, badge, and modal components out of the box. No custom JS or CSS needed.

## Common Pitfalls

### Pitfall 1: uiOutput in Module UI Root Creates Visible Page Content
**What goes wrong:** `uiOutput(ns("import_history"))` in `mod_bulk_import_ui()` renders as page content, not modal content
**Why it happens:** Shiny renders `uiOutput` wherever the UI function is called — in this case, the search notebook layout
**How to avoid:** Move history rendering inside the modal dialog footer via `renderUI` conditional on `show_history` parameter
**Warning signs:** History cards appear at bottom of page, outside modal, with no styling constraints

### Pitfall 2: Forgetting to Remove Delete Handlers When show_history = FALSE
**What goes wrong:** Delete observers run even when history UI isn't visible, wasting reactive cycles
**Why it happens:** Observer pattern in R/mod_bulk_import.R:877-896 always executes regardless of UI state
**How to avoid:** Wrap delete observer block in `if (show_history) { ... }` guard
**Warning signs:** No visible bug, but unnecessary database queries and reactive invalidations

### Pitfall 3: History Count Shows 0 When Runs Exist
**What goes wrong:** `nrow(runs)` in toggle text always shows 0
**Why it happens:** Reactive dependency not triggered before count calculation
**How to avoid:** Ensure `history_refresh()` reactive is called before `get_import_runs()` to trigger re-render
**Warning signs:** Toggle text shows "Past Imports (0)" even after successful imports

### Pitfall 4: Collapse Target ID Not Namespaced
**What goes wrong:** `data-bs-target = "#import_history_body"` conflicts when multiple module instances exist
**Why it happens:** Forgot to namespace the collapse target ID with `ns()`
**How to avoid:** Always use `paste0("#", ns("import_history_body"))` for `data-bs-target`
**Warning signs:** Clicking toggle in one module instance affects another instance

### Pitfall 5: Empty State Not Rendering When No Runs Exist
**What goes wrong:** History section completely disappears when `nrow(runs) == 0`
**Why it happens:** Original code returns `NULL` for empty history (line 844: `if (nrow(runs) == 0) return(NULL)`)
**How to avoid:** Change to render "No imports yet" message instead of `NULL`
**Warning signs:** Toggle section doesn't appear until first import run is created

### Existing Import History Render Logic (To Be Relocated)
```r
# Source: R/mod_bulk_import.R lines 838-873
output$import_history <- renderUI({
  history_refresh()  # Trigger on refresh
  nb_id <- notebook_id()
  req(nb_id)

  runs <- tryCatch(get_import_runs(con(), nb_id), error = function(e) data.frame())
  if (nrow(runs) == 0) return(NULL)  # CHANGE: Return empty message instead

  run_items <- lapply(seq_len(nrow(runs)), function(i) {
    run <- runs[i, ]
    run_date <- if (!is.null(run$created_at)) format(run$created_at, "%Y-%m-%d %H:%M") else ""
    tags$div(
      class = "border rounded p-2 mb-2",
      tags$div(
        class = "d-flex justify-content-between align-items-start",
        tags$div(
          tags$strong(class = "small", run$name),
          tags$div(class = "text-muted small", run_date)
        ),
        # DELETE BUTTON REMOVED
      ),
      tags$div(
        class = "d-flex gap-2 mt-1 small",
        tags$span(class = "text-success", paste(run$imported_count, "imported")),
        if (run$failed_count > 0) tags$span(class = "text-danger", paste(run$failed_count, "failed")),
        if (run$skipped_count > 0) tags$span(class = "text-muted", paste(run$skipped_count, "skipped"))
      )
    )
  })

  do.call(tagList, run_items)
})
```
