# Phase 53.1: Import Run Dialog Appearing at Bottom of Abstract Notebook - Context

**Gathered:** 2026-03-10
**Status:** Ready for planning

<domain>
## Phase Boundary

Fix a bug where the import run history UI renders as visible, unstyled content at the bottom of the notebook view. The `mod_bulk_import_ui()` function outputs a `uiOutput(ns("import_history"))` that renders import run cards directly into the page flow instead of being contained within the import modal. This has been present since Phase 35 (commit `650cc10`) and was never addressed. Two instances exist: sidebar import (`app.R:229`) and search notebook (`mod_search_notebook.R:360`).

</domain>

<decisions>
## Implementation Decisions

### History location
- Move import history INTO the existing import modal dialog as a collapsible section
- Place at the bottom of the import modal — below the DOI input area
- Collapsed by default — toggle text: "Past Imports (N)" with count badge
- Show last 5 import runs only (most recent first)

### History visibility
- Only the search notebook instance shows history (not the sidebar import)
- When history is empty (no prior imports), still show the section with "No imports yet" message
- Count badge visible even when collapsed: "Past Imports (3)"

### Delete capability
- Remove individual run delete buttons from the history UI
- History is read-only — runs are cleaned up automatically when the notebook is deleted
- Remove the delete observer handlers from the server accordingly

### Module API changes
- Add `show_history = FALSE` parameter to `mod_bulk_import_ui(id, show_history = FALSE)`
- Search notebook passes `show_history = TRUE`, sidebar passes `FALSE` (default)
- Also parameterize `mod_bulk_import_server` with `show_history` — when FALSE, skip history `renderUI` and delete observers entirely (no dead code)
- When `show_history = FALSE`, `mod_bulk_import_ui` outputs only the JS script tag (empty visible UI)

### Claude's Discretion
- Exact collapsible section styling (Bootstrap collapse vs custom toggle)
- History card layout within the modal (compact vs current bordered style)
- Whether to use `hr()` separator before the history section
- How to render the "No imports yet" empty state (text style, icon)

</decisions>
