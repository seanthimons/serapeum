**FILT-02: Add influential paper identification + bridge detection**

5. Add helper function `compute_trim_ids(nodes, edges)` INSIDE the server function (or as a local function before the server logic). This function:
   - Seeds are always kept: `seed_ids <- nodes$id[nodes$is_seed]`
   - Compute adaptive citation threshold:
     - For < 20 nodes: return all IDs (no trim)
     - For 20-49 nodes: 50th percentile of `cited_by_count`
     - For 50+ nodes: 75th percentile of `cited_by_count`
     - Mark threshold choice with `#NOTE: Adaptive percentile threshold — tuneable parameter`
   - Influential = seeds + papers with `cited_by_count >= threshold`
   - Bridge detection: For each non-influential paper, check if it has edges connecting TO at least one influential paper AND edges connecting FROM at least one influential paper. If so, it's a bridge. Use `edges$from` and `edges$to` (vis.js column names, NOT `from_paper_id`/`to_paper_id`).
     - Mark with `#NOTE: Bridge detection — simplified edge-based approach for citation networks (mostly DAGs). For dense graphs, consider igraph::articulation_points() instead.`
   - For networks > 500 nodes, SKIP bridge detection (too expensive, user expects aggressive pruning): just return seeds + influential.
   - Return `list(keep_ids = unique(c(seed_ids, influential_ids, bridge_ids)), remove_count = ...)`
