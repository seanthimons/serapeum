## Architecture Patterns

### Database Schema Verification Pattern
**What:** Verify table/column names against actual schema before writing queries
**When to use:** When working with recently added tables (v7.0+ bulk import tables)
**Example:**
```r
# WRONG: Assumes table name from memory
imported <- dbGetQuery(con_val, "SELECT work_id FROM import_items WHERE import_run_id = ?", list(run_id))

# CORRECT: Verified against schema in db.R:224
# Table: import_run_items (not import_items)
# FK column: run_id (not import_run_id)
# No work_id column exists — query should use doi or check citation_audit_results
imported <- dbGetQuery(con_val, "SELECT doi FROM import_run_items WHERE run_id = ? AND status = 'imported'", list(run_id))
```

### Defensive File I/O Pattern
**What:** Guard against race conditions in async file operations
**When to use:** Reading progress files written by mirai workers
**Example:**
```r
# WRONG: Assumes readLines always returns content
line <- readLines(progress_file, n = 1, warn = FALSE)
parts <- strsplit(line, "\\|")[[1]]  # Crash if line is character(0)

# CORRECT: Guard against empty file
line <- tryCatch(readLines(progress_file, n = 1, warn = FALSE),
                 error = function(e) "0|1|0|0|Waiting...")
if (length(line) == 0) return(default_progress)  # Empty file race condition
parts <- strsplit(line, "\\|", fixed = FALSE)[[1]]
if (length(parts) < 5) return(default_progress)  # Partial write race condition
```

### OpenAlex Work ID Fallback Pattern
**What:** Store Work ID alongside DOI, retry failed DOI lookups with Work ID
**When to use:** Abstract re-lookup operations (seed discovery, citation network node clicks)
**Example:**
```r
# Storage: abstracts table already has paper_id (Work ID) + doi column
# Both are populated from parse_openalex_work() in api_openalex.R:146-148

# Lookup with fallback:
fetch_paper <- function(doi = NULL, work_id = NULL, email, api_key) {
  result <- NULL

  # Try DOI first if provided
  if (!is.null(doi)) {
    result <- tryCatch(
      fetch_work_by_id(doi, email, api_key),
      error = function(e) NULL
    )
  }

  # Silently fallback to Work ID if DOI failed
  if (is.null(result) && !is.null(work_id)) {
    result <- tryCatch(
      fetch_work_by_id(work_id, email, api_key),
      error = function(e) NULL
    )
  }

  result
}
```
**Note:** `fetch_work_by_id()` in `api_openalex.R:487` already handles both DOI URLs and Work IDs via URLencode — no API changes needed, just add retry logic in callers.

### visNetwork Physics Control Pattern
**What:** Use visNetworkProxy to toggle physics on existing graph
**When to use:** Runtime control of network simulation without rebuilding graph
**Example:**
```r
# UI: bslib switch in legend panel
input_switch(
  ns("physics_enabled"),
  "Physics Simulation",
  value = TRUE
)

# Server: observe switch, update via proxy
observeEvent(input$physics_enabled, {
  req(current_network_data())  # Only update if network exists
  visNetwork::visNetworkProxy("network_graph") |>
    visNetwork::visPhysics(enabled = input$physics_enabled)
}, ignoreInit = TRUE)
```
**Source:** vis-network API supports `setOptions({physics: {enabled: true/false}})` — visNetworkProxy wraps this via `visPhysics()` method.


### Anti-Patterns to Avoid
- **Blind table name assumptions:** Always verify against `db.R` schema for tables added in recent phases
- **Unguarded file reads:** Async file operations need defensive coding against empty/partial writes
- **DOI-only lookups:** OpenAlex Work IDs are more reliable than DOIs for re-lookup operations
- **Rebuilding graph for settings:** Use visNetworkProxy for runtime updates, not full graph regeneration

## Common Pitfalls

### Pitfall 1: DuckDB Foreign Key Column Naming
**What goes wrong:** Assuming FK column names match referenced table (e.g., `import_run_id` when table uses `run_id`)
**Why it happens:** DuckDB doesn't enforce FK naming conventions — developers choose column names
**How to avoid:** Always read `CREATE TABLE` statement in `db.R` for exact column names
**Warning signs:** `Error: column "X" does not exist` in queries that look syntactically correct

### Pitfall 2: Race Conditions in Async File I/O
**What goes wrong:** `readLines()` returns `character(0)` when file exists but is empty/incomplete
**Why it happens:** Mirai worker hasn't finished writing, or write is interrupted
**How to avoid:** Guard with `length() == 0` check before subscripting, provide default value
**Warning signs:** `Error: subscript out of bounds` on `[[1]]` indexing of strsplit result

### Pitfall 3: DOI Display vs Canonical Mismatch
**What goes wrong:** OpenAlex returns display DOI (e.g., `10.65215/2q58a426`) but canonical DOI in `ids.doi` differs
**Why it happens:** Publishers register multiple DOIs, OpenAlex prefers display DOI but API lookup requires canonical
**How to avoid:** Store Work ID (always unique) alongside DOI, fallback to Work ID on lookup failure
**Warning signs:** User reports "paper not found" for papers that exist in OpenAlex, DOI lookup returns 404

### Pitfall 4: visPhysics Not Triggering
**What goes wrong:** `visPhysics(enabled = ...)` called but graph doesn't respond
**Why it happens:** Called before network data exists, or proxy targets wrong network ID
**How to avoid:** Guard with `req(current_network_data())`, verify proxy ID matches renderVisNetwork output ID
**Warning signs:** observeEvent fires (console shows execution) but graph behavior unchanged

### Pitfall 5: Duplicate renderUI Calls
**What goes wrong:** Same UI section renders in multiple locations
**Why it happens:** Copy-paste of `output$X <- renderUI()` without removing original
**How to avoid:** Search codebase for all `output${output_id}` before adding new renderUI
**Warning signs:** User sees same content twice, UI elements have identical IDs (breaks Shiny reactivity)

### Fix Progress File Race Condition
```r
# Location: R/bulk_import.R:127-129
# WRONG (current code):
line <- tryCatch(readLines(progress_file, n = 1, warn = FALSE),
                 error = function(e) "0|1|0|0|Waiting...")
parts <- strsplit(line, "\\|", fixed = FALSE)[[1]]  # Crash if line is character(0)

# CORRECT:
line <- tryCatch(readLines(progress_file, n = 1, warn = FALSE),
                 error = function(e) "0|1|0|0|Waiting...")

# Guard against empty file (race condition)
if (length(line) == 0 || nchar(line[1]) == 0) {
  return(default)
}

parts <- strsplit(line, "\\|", fixed = FALSE)[[1]]

# Guard against partial write (another race condition)
if (length(parts) < 5) {
  return(default)
}
```

### OpenAlex Work ID Fallback (DOI Re-lookup)
```r
# Storage: abstracts table has both paper_id (Work ID) and doi
# Example: a paper with paper_id="W2626778328", doi="10.5555/3295222.3295349"

# Fetch paper with fallback (add to api_openalex.R or caller):
fetch_paper_with_fallback <- function(doi = NULL, paper_id = NULL, email, api_key) {
  # Try DOI first if provided
  if (!is.null(doi)) {
    result <- tryCatch({
      fetch_work_by_id(doi, email, api_key)
    }, error = function(e) NULL)

    if (!is.null(result)) return(result)
  }

  # Silently fallback to Work ID
  if (!is.null(paper_id)) {
    result <- tryCatch({
      fetch_work_by_id(paper_id, email, api_key)
    }, error = function(e) NULL)

    if (!is.null(result)) return(result)
  }

  # Both failed
  stop("Paper not found via DOI or Work ID")
}

# Usage in seed discovery or citation network node click:
# 1. Query DB for both doi and paper_id
# 2. Call fetch_paper_with_fallback(doi = row$doi, paper_id = row$paper_id, ...)
# 3. User never sees "not found" error for papers that exist
```
