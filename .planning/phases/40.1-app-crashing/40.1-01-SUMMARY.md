## Technical Implementation

### Fix 1: Seed Network SQL (mod_bulk_import.R)

**Before:**
```r
imported <- dbGetQuery(con_val, "SELECT work_id FROM import_items WHERE import_run_id = ? AND status = 'imported'", list(run_id))
network_seed_request(list(seed_ids = imported$work_id, ...))
```

**After:**
```r
imported <- dbGetQuery(con_val, "
  SELECT a.paper_id
  FROM import_run_items i
  JOIN abstracts a ON i.doi = a.doi
  WHERE i.run_id = ? AND i.status = 'imported'
", list(run_id))
network_seed_request(list(seed_ids = imported$paper_id, ...))
```

**Why:** The actual schema uses `import_run_items` table with `run_id` FK, and stores only DOI (no `work_id` column). Must join to `abstracts` to get `paper_id` (Work ID).

### Fix 2: Progress File Race Condition (bulk_import.R)

**Before:**
```r
line <- tryCatch(readLines(progress_file, n = 1, warn = FALSE),
                 error = function(e) "0|1|0|0|Waiting...")
parts <- strsplit(line, "\\|", fixed = FALSE)[[1]]  # CRASH if line is character(0)
```

**After:**
```r
line <- tryCatch(readLines(progress_file, n = 1, warn = FALSE),
                 error = function(e) "0|1|0|0|Waiting...")

# Guard against empty file (race condition with mirai worker)
if (length(line) == 0 || nchar(line[1]) == 0) {
  return(default)
}

parts <- strsplit(line, "\\|", fixed = FALSE)[[1]]
```

**Why:** Mirai workers create the file first, then write to it. Polling can catch the file empty, causing `character(0)` from `readLines` which crashes `strsplit`.

### Fix 3: DOI-to-Work-ID Fallback (api_openalex.R, mod_seed_discovery.R)

**New function:**
```r
fetch_paper_with_fallback <- function(doi = NULL, paper_id = NULL, email, api_key = NULL) {
  # Try DOI first if provided
  if (!is.null(doi) && nchar(doi) > 0) {
    result <- tryCatch(get_paper(doi, email, api_key), error = function(e) NULL)
    if (!is.null(result)) return(result)
  }

  # Silently fallback to Work ID
  if (!is.null(paper_id) && nchar(paper_id) > 0) {
    result <- tryCatch(get_paper(paper_id, email, api_key), error = function(e) NULL)
    if (!is.null(result)) return(result)
  }

  NULL
}
```

**Usage in seed discovery:**
```r
# Try to find Work ID in DB for fallback
db_paper <- tryCatch(
  DBI::dbGetQuery(con(), "SELECT paper_id FROM abstracts WHERE doi = ? LIMIT 1", list(doi)),
  error = function(e) data.frame()
)
work_id <- if (nrow(db_paper) > 0) db_paper$paper_id[1] else NULL

paper <- tryCatch({
  fetch_paper_with_fallback(doi = doi, paper_id = work_id, email = email, api_key = api_key)
}, error = function(e) { ... })
```

**Why:** Some papers in OpenAlex have display DOIs that differ from their canonical DOIs. User imports paper with DOI A (canonical in OpenAlex), tries to seed from it using DOI B (display DOI in citation), lookup fails. Fallback to Work ID (which we stored in DB during import) solves this.
