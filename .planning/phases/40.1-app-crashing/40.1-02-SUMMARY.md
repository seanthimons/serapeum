## Decisions Made

- **Silent auto-lookup:** When DOI is pre-filled via "Use as Seed" button, the lookup happens automatically without showing a notification. The paper preview appearing is sufficient feedback. Manual lookup via "Look Up" button still shows "Paper found!" notification.
- **View switch as feedback:** When seeding citation network from search notebook or bulk import, switching to the network view is sufficient feedback. No toast notification needed since the user sees the view change and seeds are loaded.
- **Module namespacing:** visNetworkProxy calls inside Shiny modules MUST use `session$ns("output_id")` not bare `"output_id"` strings. This was a critical bug causing the physics toggle to fail silently.

## Deviations from Plan

Plan specified two fixes (physics toggle relocation + imported papers duplication). User testing revealed four additional issues after checkpoint feedback.

### Auto-fixed Issues

**1. [Rule 1 - Bug] Physics toggle not working**
- **Found during:** Task 2 (checkpoint:human-verify user feedback)
- **Issue:** Physics toggle in legend panel did not freeze/unfreeze nodes when toggled
- **Root cause:** visNetworkProxy calls at lines 745 and 756 used bare string `"network_graph"` instead of `session$ns("network_graph")`, so proxy calls targeted wrong ID
- **Fix:** Added `session$ns()` wrapper to both visNetworkProxy calls (physics observer and palette observer)
- **Files modified:** R/mod_citation_network.R
- **Verification:** Module loads without errors
- **Committed in:** `225768e`

**2. [Rule 1 - Bug] Button layout unbalanced**
- **Found during:** Task 2 (checkpoint:human-verify user feedback)
- **Issue:** Build Network and Save Network buttons not vertically aligned after removing physics checkbox from build row
- **Root cause:** Buttons were bare elements without vertical centering containers
- **Fix:** Wrapped both buttons in `div(class = "d-flex align-items-center", ...)` for flexbox vertical centering
- **Files modified:** R/mod_citation_network.R
- **Verification:** Layout uses Bootstrap flexbox classes correctly
- **Committed in:** `225768e`

**3. [Rule 1 - Bug] Double "Paper found" modal**
- **Found during:** Task 2 (checkpoint:human-verify user feedback)
- **Issue:** When using "Use as Seed" flow from search notebook, user saw duplicate "Paper found" notifications
- **Root cause:** Pre-fill DOI observer (auto-lookup) showed "Seed paper loaded!" notification AND manual lookup observer showed "Paper found!" notification
- **Fix:** Removed showNotification from pre_fill_doi observer - auto-triggered lookup is now silent, paper preview appearing is sufficient feedback
- **Files modified:** R/mod_seed_discovery.R
- **Verification:** Module loads without errors
- **Committed in:** `56126e4`

**4. [Rule 1 - Bug] Double seed network notifications**
- **Found during:** Task 2 (checkpoint:human-verify user feedback)
- **Issue:** Users saw redundant notifications when seeding citation network ("Seeding network with X papers" followed by "Network built: Y papers from Z seeds")
- **Root cause:** search_notebook and bulk_import modules both showed notifications when network_seed_request was triggered, creating notification spam
- **Fix:** Removed showNotification calls from both modules - switching to network view is sufficient feedback, user will see "Network built" notification after clicking Build Network button
- **Files modified:** R/mod_search_notebook.R, R/mod_bulk_import.R
- **Verification:** Modules load without errors
