<task type="auto">
  <name>Task 2: Wire healing server logic with retry tracking and fallback</name>
  <files>R/mod_slides.R</files>
  <action>
1. **Add state fields to `generation_state` reactiveValues**: Add `heal_attempts = 0`, `validation_errors = NULL`, `is_fallback = FALSE`, `last_chunks = NULL` (to store chunks for fallback).

2. **Update `input$generate` observer**: After `generate_slides()` returns successfully:
   - Store chunks: `generation_state$last_chunks <- chunks`
   - Reset heal state: `generation_state$heal_attempts <- 0`, `generation_state$is_fallback <- FALSE`
   - Check `result$validation`: If `result$validation$valid` is FALSE, store validation errors: `generation_state$validation_errors <- result$validation$errors`
   - When showing results modal, pass validation_errors and qmd_content to `mod_slides_results_ui()`
   - When render fails (html_result$error), pass qmd_content to results modal so "Show raw output" works
   - On full success (preview rendered), pass qmd_content for the raw output toggle

3. **Add `input$open_heal` observer** (opens healing modal): Extract current errors from `generation_state$validation_errors` or `generation_state$error`. Determine `is_success` (both are NULL/empty). Call `get_healing_chips(errors, is_success)`, store in `current_chips()`. Show healing modal: `showModal(mod_slides_heal_modal_ui(ns, errors, is_success))`.

4. **Add chip click observers**: Use `observeEvent()` for each possible chip. Since chips are dynamic, use a pattern: create an observer that watches for any input matching the chip pattern. Use `lapply(seq_len(20), function(i) { observeEvent(input[[paste0("chip_", i)]], { chips <- current_chips(); if (i <= length(chips)) updateTextAreaInput(session, "heal_instructions", value = chips[i]) }, ignoreInit = TRUE) })` — observe up to 20 chip slots. When a chip is clicked, auto-fill the text input with the chip label.

5. **Add `input$do_heal` observer** (executes healing): This is the core healing logic:
   ```
   a. Increment heal_attempts: generation_state$heal_attempts <- generation_state$heal_attempts + 1
   b. Get current attempt: attempt <- generation_state$heal_attempts
   c. If attempt > 2: FALLBACK PATH
      - Get chunks from generation_state$last_chunks
      - Get notebook name
      - Call build_fallback_qmd(chunks, notebook_name)
      - Set generation_state$is_fallback <- TRUE
      - Save to temp file, set qmd_path
      - Render with Quarto, show results modal with is_fallback = TRUE
      - Return
   d. HEALING PATH (attempt <= 2):
      - Close modal, show loading results modal
      - Get previous QMD from generation_state$qmd_content
      - Get errors from generation_state$validation_errors or generation_state$error
      - Get instructions from input$heal_instructions
      - Get API key and model from config/last_options
      - Call heal_slides(api_key, model, previous_qmd, errors, instructions, con(), session$token)
      - If heal_slides returns error: show results modal with error + heal_attempts
      - If success: validate with validate_qmd_yaml(), store new qmd_content
        - If validation fails: store validation_errors, show results modal with error panel
        - If validation passes: inject theme + CSS, render HTML, show preview
   ```

6. **Update `input$regenerate` observer**: Reset healing state: `generation_state$heal_attempts <- 0`, `generation_state$is_fallback <- FALSE`, `generation_state$validation_errors <- NULL`. Then show config modal (existing behavior).

7. **Ensure results modal always passes current state**: Every call to `mod_slides_results_ui()` throughout the server should pass `qmd_content = generation_state$qmd_content`, `validation_errors = generation_state$validation_errors`, `heal_attempts = generation_state$heal_attempts`, `is_fallback = generation_state$is_fallback`.
