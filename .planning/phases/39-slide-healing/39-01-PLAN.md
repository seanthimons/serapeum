<task type="auto">
  <name>Task 1: Improve system prompt and add validation + fallback functions</name>
  <files>R/slides.R</files>
  <action>
1. **Improve `build_slides_prompt()` system prompt** (SLIDE-01): Replace the current system_prompt in `build_slides_prompt()` with an improved version that includes an explicit YAML template structure. The new system prompt should:
   - Include a verbatim YAML frontmatter template showing exact structure (title, format/revealjs)
   - Emphasize "CRITICAL: Your output must start with valid YAML frontmatter"
   - Add rules: "Ensure all YAML is properly indented with spaces (not tabs)", "Do not include any content before the opening ---"
   - Keep existing rules about ## for slides, # for sections, max 5-7 bullet points, speaker notes, no code fences

2. **Add `validate_qmd_yaml(qmd_content)` function**: Extracts YAML frontmatter between `---` delimiters using regex `^---\n(.*?)\n---` (with `perl = TRUE` for non-greedy). Parses with `yaml::yaml.load()` inside `tryCatch`. Returns `list(valid = TRUE/FALSE, errors = character(), parsed = list/NULL)`. Handle edge cases:
   - No `---` delimiters found: return error "No YAML frontmatter found (missing --- delimiters)"
   - yaml parse error: capture error message (includes line/column info)
   - Empty frontmatter: return error "Empty YAML frontmatter"

3. **Add `build_healing_prompt(previous_qmd, errors, instructions)` function**: Creates system + user prompt for targeted healing. System prompt: "You are an expert Quarto presentation fixer. You receive a broken or imperfect .qmd file and specific instructions on what to fix. Make targeted changes while preserving the parts that work. Output ONLY the complete fixed .qmd content. No explanations, no code fences. Ensure YAML frontmatter is valid." User prompt includes the full previous QMD in a code block, validation errors (if any), and user instructions. Returns `list(system, user)`.

4. **Add `heal_slides(api_key, model, previous_qmd, errors, instructions, con, session_id)` function**: Calls `build_healing_prompt()` to get prompts, then `format_chat_messages()` + `chat_completion()` (same pattern as `generate_slides()`). Cleans up response (strip code fences - same gsub as generate_slides). Logs cost if con/session_id provided (same pattern as generate_slides). Saves to temp file. Returns `list(qmd, qmd_path, error)`.

5. **Add `build_fallback_qmd(chunks, notebook_name)` function**: Generates a minimal valid QMD template. Structure:
   - Valid YAML frontmatter with title (from notebook_name) and format: revealjs
   - "## Overview" slide with bullet point listing number of source documents
   - For each unique `doc_name` in chunks: a "## {doc_name without extension}" slide with the first line of the first chunk for that document (truncated to 100 chars) as a bullet point
   Returns the QMD string.

6. **Add `get_healing_chips(errors, is_success)` function**: Returns a character vector of chip labels. If `is_success` is TRUE: return cosmetic chips c("Fewer bullet points", "Make text bigger", "Add more detail", "Shorten content", "Simplify slides"). If `is_success` is FALSE: start with error-specific chips (check if errors contain "YAML|parse|syntax" -> "Fix YAML syntax", "CSS|style|format" -> "Fix CSS formatting", "render|quarto" -> "Fix Quarto formatting") then append baseline chips c("Simplify slides", "Fewer bullet points"). Use `grepl(..., ignore.case = TRUE)` for matching.

7. **Update `generate_slides()` to call `validate_qmd_yaml()`** after code fence cleanup but before theme/CSS injection. If validation fails, still return the QMD content but add a `validation` field to the return list: `list(qmd, qmd_path, error = NULL, validation = list(valid, errors))`. If validation passes, `validation = list(valid = TRUE, errors = character(0))`. This lets the UI decide what to show.
