# Phase 5: Sublation (Round 1) — The Persistence Hunter Architecture

## Cancellation
- **A is wrong** that verifiability alone redeems autonomy. A permanently capable auditor is assumed, but the system erodes the competence auditing requires.
- **B is wrong** that all relevance judgment is constitutive. The judgment stack is not monolithic — scanning for keyword relevance is not the same as recognizing cross-disciplinary connections.

## Preservation
- **From A:** Scale is real. Opaque autonomy is strictly worse than transparent autonomy. Verifiability is necessary (but not sufficient).
- **From B:** Deskilling is real (Karpathy confirms). Automation bias is documented. Generation and discrimination are different — something IS lost.

## Elevation: The Persistence Hunter Architecture

A recursive research agent is a persistence hunting partner — an endurance system paired with a tracker.

The agent is the **endurance layer**: tireless citation-chain following, abstract scanning, state maintenance across branching paths.

The researcher is the **tracker**: called to the trail at **critical junctures** — moments of genuine ambiguity where the path forks and the choice requires understanding the agent cannot replicate.

This is NOT:
- "Human reviews everything at the end" (QA, not research)
- "Human does everything themselves" (nostalgia at 400M scale)
- "Use autonomy for some things, human for others" (division of labor, not sublation)

This IS: **the system identifies where judgment matters and surfaces only those moments.**

## Architectural Requirements

1. **Visible omissions** — show pruned branches (close calls), not just selections
2. **Serendipity injection** — anti-pruning: follow unexpected branches architecturally
3. **Calibration loops** — periodic prediction-before-reveal to maintain generative capacity
4. **Expertise-adaptive autonomy** — surface more forks for novices, fewer for experts, maximize autonomy for the median user
5. **The Compass, not the Map** — human defines direction; agent proposes course corrections at evidence-based moments

## The Dune Quote Reconsidered

The persistence hunter is not "a machine in the likeness of a human mind." It's a machine in the likeness of human *endurance*. The human mind remains the tracker. The machine that removed the Dune quote lacked this architecture — it had no concept of "critical junctures." The lesson: build the forks into the architecture.

## Model Update
- **Before:** Should we automate research judgment? (Binary yes/no)
- **After:** Research judgment is a stack. Architecture must identify which layers are endurance (automatable) and which are tracking (human-essential), and adapt based on the user.
- **Because:** Both monks treated judgment as monolithic. Karpathy's generation/discrimination split, persistence hunting, and the median-user intervention revealed the real question: not *whether* but *at what layer* and *for whom*.

## New Contradictions Generated

1. **The Calibration Paradox:** Calibration friction vs. the core value proposition of saving time
2. **The Serendipity Problem:** Machine-selected "surprise" is bounded by the machine's model of surprise
3. **The Slopacolypse Feedback Loop:** Autonomous agents generate more output → more noise → more need for agents → more noise
